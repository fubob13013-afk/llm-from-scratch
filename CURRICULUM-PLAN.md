# 大模型训练原理 · 全新课程大纲

> **状态：✅ 全部完成（2026-07-04）** — 此文档为 v2.0 重构方案的历史记录
> 
> 设计原则：一个概念 → 具体例子 → 交互原型
> 目标读者：零基础、高中数学水平、对电脑一知半解
> 最终目标：理解大模型训练原理，能上手训练自己的小模型

---

## 当前 7 节课的问题诊断

| 课次 | 当前内容 | 核心问题 |
|------|---------|---------|
| L1 | LLM 完整生命周期（预训练→微调→对齐→部署） | **太广**：一次塞了四个阶段，应该是"课程地图"而非第一课 |
| L2 | 分词 + BPE + Token ID + 滑动窗口 + DataLoader | **略密**但结构好，BPE 演示出色 |
| L3 | Embedding + 位置编码 preview | 位置编码被"预览"而非完整讲授，后续 L7 才展开——碎片化 |
| L4 | Q/K/V + 多头 + 因果遮罩 | **多头注意力**让单头的理解还没巩固就跳到复杂变体 |
| L5 | FFN + 残差 + LayerNorm + Block 概念 + 导数概念 | **最混乱**：4 个独立概念 + 数学前置知识全塞一堂课 |
| L6 | 损失函数 + 梯度下降 + 训练循环 | 内容好，但**导航 bug**（右侧链接指向学习记录而非 L7） |
| L7 | 位置编码 + 完整架构 + 自回归生成 | 位置编码应该在 L3 完整讲授 |

## 新架构：7 课递进路线

```
L1: LLM 做什么？      → 核心任务：预测下一个 token（动机建立）
L2: 文本怎么变数字？   → 分词 + BPE + Token ID + 训练样本（数据流水线）
L3: 数字怎么带语义？   → Embedding + 位置编码（从数字到含义的桥梁）
L4: 词语怎么交流？     → 注意力机制：Q/K/V + 因果遮罩（单头，聚焦核心）
L5: 一层处理怎么做？   → 多头注意力 + FFN + 残差 + LayerNorm（完整 Block）
L6: 模型怎么学习？     → 损失函数 + 梯度下降 + 训练循环（优化过程）
L7: 怎么生成新文本？   → 完整架构串联 + 自回归解码 + 采样策略（收官）
```

### 每课结构模板

```
1. 前置提醒（需要上一课的什么知识）
2. 核心问题（本课回答什么问题）
3. 概念讲解（从直觉到精确定义）
4. 具体例子（中文示例，手动走一遍）
5. 交互原型（浏览器内可操作的实验）
6. 小测验（3 道检索练习）
7. 推荐资源（B站视频等）
8. 导航（← 上一课 | 下一课 →）
```

### 关键改动

| 改动 | 原因 |
|------|------|
| L1 不再讲完整生命周期 | 第一课应建立核心心智模型（预测下一个token），生命周期移到末尾作为"延伸阅读" |
| L3 合并位置编码 | 消除"preview → 3课后才讲"的碎片化 |
| L4 移除多头注意力 | 让学习者先彻底理解单头，L5 再自然引出"一个头不够" |
| L5 纳入多头注意力 | "为什么需要多头"成为 Block 设计的自然动机之一 |
| L6 修复导航 | 右侧链接指向 L7 |
| L7 移除位置编码 | 位置编码已在 L3 完整讲授，L7 聚焦组装与生成 |

### 概念依赖图

```
L1 (预测任务)
 └─ L2 (数据准备)
     └─ L3 (Embedding + 位置编码)
         └─ L4 (注意力机制)
             └─ L5 (Transformer Block)
                 └─ L6 (训练循环)
                     └─ L7 (组装与生成)
```

每课只依赖前一课，不跳跃、不预览、不事后修补。
