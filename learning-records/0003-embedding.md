# Embedding：从 Token ID 到语义坐标

Bob 完成了第 3 课，理解了 Embedding 层的核心机制。

## 课前状态
- 已理解 Token ID 是整数编号，但不清楚这些整数如何变成模型能计算的东西
- 对「向量」概念陌生，需要从零建立直觉

## 核心理解（课后）
- **Token ID 的问题**：纯粹的编号，ID=15 和 ID=16 的「距离」是人为分配的，不反映语义关系
- **Embedding = 查表**：一个 (词表大小 × 嵌入维度) 的巨大表格，输入 Token ID，查出对应行的一串数字（向量）
- **向量的直觉**：就像 RGB 颜色空间——(230,57,70) 和 (230,115,70) 是相近的红色系，(69,123,157) 是蓝色系。Embedding 也是这个逻辑，只是有 768 维而不是 3 维
- **为什么能行**：分布式语义假说——出现在相似上下文的词，训练中自然被调到彼此附近
- **位置嵌入**：同一词在不同位置的词嵌入是一样的，需要额外叠加位置信息才能区分「狗追猫」和「猫追狗」

## 有效类比（来自本课）
- RGB 颜色空间：Bob 的设计背景让他对颜色类比吸收极快
- 画布尺寸：A6 vs A4 vs 壁画，解释嵌入维度的 trade-off

## 遗留问题
- 位置嵌入的具体算法（正弦编码 vs 可学习）—— 留到 Transformer 课
- 嵌入向量在训练中具体怎么「调」—— 涉及反向传播，留到训练课
- 「国王 - 男人 + 女人 = 王后」这种语义运算的具体原理

## 教学观察
- Bob 的设计背景是巨大优势——颜色空间类比立刻打通了「高维向量」的直觉
- 表格可视化（查表过程）比纯文字有效得多
- 应该在第 2 课和第 3 课之间建立更明确的连接——重申「从 Token ID 到 Embedding」是整个流程的连续步骤
