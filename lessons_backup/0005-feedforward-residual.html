🧱 第5课 · 为什么 Attention 做一次不够？—— Transformer Block 的完整结构

第 4 课讲完了 Attention。一个自然的问题是：做完一次 Attention，然后呢？本课回答这个问题，并补完 Transformer 中除了 Attention 之外的所有结构。

⚠️ 本课前提 需要知道 Attention 的输入和输出都是向量序列（第 4 课）。具体说：输入是 n 个 token 各自的一个向量（来自 Embedding），Attention 让它们互相「看」了一遍，输出 n 个融合了上下文信息的新向量。

起点：做完一次 Attention 之后，得到了什么？

假设输入句子是「我 喜欢 猫」。经过第 4 课讲的 Attention 之后：

  • 「我」的向量里融入了「喜欢」和「猫」的信息
  • 「喜欢」的向量里融入了「我」（主语）和「猫」（宾语）的信息
  • 「猫」的向量里融入了「我」和「喜欢」的信息

三个词不再是孤立的——每个都携带了整句话的上下文。这已经比原始的 Embedding 向量有用得多了。

但一个问题出现了：这就够了吗？

「我喜欢猫」是一个极短的句子。真实文本中，一个段落可能有几百个 token，涉及多层嵌套的指代关系、长距离的语义依赖。只做一次 Attention，每个词只能捕获到和它直接相邻的信息——因为它的 Q 只和原始向量的 K 做了一次匹配。复杂的语义关系往往需要多次「传递」才能建立。

更具体地说：在只有一层 Attention 的情况下，第 1 个词和第 100 个词建立联系的唯一途径是它们之间的点积得分足够高。但在复杂文本中，两个远距离相关的词往往需要中间词来「桥接」。每一层 Attention 可以让信息在相邻位置之间流动一次，多层堆叠之后，信息可以逐层传递到更远的位置。

所以，注意力（Attention）需要重复做。不止一次，而是很多次。

⚠️ 一个容易误解的点：重复做 Attention 不是同一个操作机械地重复
如果你把同一个 Attention（参数相同、输入相同）重复算 12 次，结果不会变——就像把同一张纸复印 12 次还是同一张纸。

实际情况是：每一层有自己独立的 Attention 参数。Block₁ 的 WQ, WK, WV 和 Block₂ 的 WQ, WK, WV 是两套完全不同的数字，各自在训练中学到了不同的匹配规则。

而且每一层的输入也不同：Block₁ 的输入是原始 Embedding 向量；Block₁ 加工完后输出新的向量给 Block₂。Block₂ 拿到的输入已经是 Block₁ 处理过的、融合了第一轮上下文的表示。所以即使参数相同（实际上也不同），输入不同也决定了输出不同。

概括：不是「同一个 Attention 做了 12 遍」，而是「12 个不同的 Attention 层串联，每一层在上层的基础上继续精炼」。

什么是「Block」（块）？—— 神经网络的基本建筑单元

在深入 Transformer 的具体设计之前，先理解一个更基础的概念：在深度学习中，「Block」（块，有时也叫 Layer / 层）就是一个可复用的处理单元。

它的工作方式很简单：

输入向量 → [ Block ] → 输出向量

你设计好一个 Block 的内部结构，然后把它复制 N 份，首尾相连叠在一起：

输入 → [ Block₁ ] → [ Block₂ ] → [ Block₃ ] → …… → [ Blockₙ ] → 输出

几个关键点：

  • 所有 Block 的结构相同（比如都是「先 Attention 再 FFN」），但参数独立（Block₁ 的权重矩阵和 Block₂ 的不是同一套数字，各自学习）
  • 数据依次流过每一个 Block。Block₁ 的输出是 Block₂ 的输入，以此类推
  • 叠得越多，模型能捕捉的模式就越复杂。浅层 Block 倾向学到局部、低级的模式（哪些词经常共现），深层 Block 倾向学到全局、高级的模式（段落主旨、逻辑关系）

Transformer Block 就是这个框架下的一个具体设计——每个 Block 内部包含 Attention 和一些配套组件。GPT-2 小型叠了 12 个，GPT-3 叠了 96 个。

一个 Transformer Block（Transformer 块）需要完成哪些事？

设计一个 Block，本质上是在回答：「一轮完整的处理」应该包含哪些步骤？

拆开来看，有两个功能层面的需求，和两个工程层面的需求：

类型需求对应组件
功能层面 让不同位置的 token 交换信息 Multi-Head Attention（第 4 课）
功能层面 对每个 token 的信息做非线性加工 Feed-Forward Network（本课 §1）
工程层面 保证堆了很多层之后梯度不消失 残差连接（本课 §2）
工程层面 防止数值范围在多层之后漂移失控 Layer Normalization（本课 §3）

接下来逐个解释后三个——它们分别解决什么问题，以及具体怎么做。

§1 Feed-Forward Network（FFN）—— 逐位置的非线性加工

它解决什么问题

注意力（Attention）的输出是多个 Value 向量的加权平均。加权平均有一个本质局限：它只能产生输入向量的线性组合。

什么是线性？什么是非线性？

一个变换（变换 = 把输入数字经过某种规则变成输出数字）是线性的，意味着画在坐标系里是一条直线。线性变换只做三件事：拉伸、旋转、平移。

线性：  y = 2x + 3    → 画出来是直线
         y = 0.5x      → 画出来是直线

线性变换有一个关键限制：如果一组数据在输入空间里排成一条直线，线性变换后它们还是一条直线。它只能改变直线的方向和位置，不能把直线「掰弯」。

非线性变换就是画出来不是直线的变换：

非线性：y = x²         → 画出来是抛物线（弯的）
        y = max(0, x)  → 画出来在 x=0 处有一个折角
        y = 1/(1+e⁻ˣ)  → 画出来是 S 形曲线

非线性变换可以改变数据的形状——直线进去，曲线出来。

这和 Attention 有什么关系？

Attention 做加权平均 → 线性操作。这意味着 Attention 的输出永远被限制在输入向量的线性组合范围内——它不能创造出性质完全不同的新表示。

举例：假设 Attention 从上下文中聚合了「不」和「好看」两个词的向量。对它们做加权平均，得到的是「不」和「好看」的中间状态——它既有「不」的属性又有「好看」的属性，但它不是「丑」。要得到「不好看 ≈ 丑」这个语义，你需要一个非线性变换来把「不 + 好看」的组合映射到一个完全不同的方向。

这就是 FFN（Feed-Forward Network，前馈网络）存在的理由：引入非线性，让模型能表示线性组合无法表达的关系。否定、条件、转折、类比——这些语言中的核心逻辑关系，都是非线性的。

具体结构

FFN(x) = GELU( x · W₁ + b₁ ) · W₂ + b₂

其中：

  • x：一个 token 的向量（768 维）
  • W₁：第一层权重矩阵，形状为 768 × 3072（将输入升维到 4 倍）
  • b₁：第一层的偏置（bias），形状为 3072 维。作用：在 x·W₁ 的拉伸/旋转之外提供一个「平移」。类比一次函数 y = kx + b 中的 b——没有它，输入为 0 时输出也必然是 0，变换被限制在穿过原点的直线上。有了 b₁，高维空间的变换才能自由平移
  • GELU：激活函数，引入非线性。输入为负时输出接近 0，输入为正时接近恒等
  • W₂：第二层权重矩阵，形状为 3072 × 768（降维回原始维度）
  • b₂：第二层的偏置，形状为 768 维。让降维后的输出也有平移自由度，最终 FFN(x) 能到达原始向量空间的任意位置

三步操作：升维（768→3072）→ 非线性激活 → 降维（3072→768）。

为什么要「升维再降维」？

中间的 3072 维给了模型一个更大的「临时计算空间」。在低维空间中线性不可分的数据，投影到高维后可能变得可分。激活函数在这个高维空间里制造非线性，然后把结果压回原始维度。这是一种用冗余换表达能力的设计。

下面有一组排成网格的点。先点「线性变换」观察：点动了，但网格线依然是直的——只是拉伸旋转。再点「+ ReLU」观察：负半轴的点被压扁到 y=0，整个形状被「掰弯」了。这就是 Attention（线性）和 FFN（非线性）的本质区别。

两个关键性质

  • 逐位置独立：FFN 对每个 token 分别计算，不同 token 之间不交换信息。交换信息是 Attention 的工作。FFN 只负责「拿到 Attention 聚合好的上下文之后，怎么加工这个 token 自己的表示」。
  • 参数共享：所有 token 位置使用同一套 W₁ 和 W₂。因为你希望对「第一个词」和「第五个词」施加同样类型的非线性变换规则。

§2 残差连接（Residual Connection）—— 保证深层网络能训练

它解决什么问题

现在我们知道了每个 Block 里要有 Attention 和 FFN 两个操作。一个直觉的做法是把它们串起来：

x → Attention(x) → FFN(Attention(x)) → 输出

这在层数少的时候没问题。但当堆到 12 层、96 层时，出现了一个严重的工程问题：反向传播时，梯度从最后一层传回第一层，每穿过一个操作就衰减一点。到了前面几层，梯度几乎为零——参数无法更新，模型学不动。

这叫梯度消失。根源在于梯度必须穿过每个 Block 内部的所有计算步骤（矩阵乘法、激活函数等），每一步都可能让梯度的模变小。

插曲：导数在训练中到底起什么作用？

要理解「梯度消失为什么是大问题」，得先理解导数在训练中的角色。

训练模型的本质，可以类比为蒙着眼睛站在一座山上，要走到山谷的最低点。你可以做一件事：用脚感受脚下的坡度——往某个方向迈一步，感觉自己是往下走了还是往上走了。这个「坡度」就是导数。

翻译成数学：模型有几十亿个参数（W₁, W₂, b₁, …），定义了一个损失函数 L（衡量模型当前表现有多差）。对某个参数求 L 的导数，就得到了它的梯度——含义是：其他参数都不动，单独把这个值调大一点点，损失会变大还是变小？

  • 导数 > 0：调大它 → 损失变大（往山上走）→ 应该减小这个参数
  • 导数 < 0：调大它 → 损失变小（往山下走）→ 应该增大这个参数

每次把每个参数往导数的反方向挪一小步：

新参数 = 旧参数 - 学习率 × 导数

对几十亿个参数都这样做一次，就是一步训练。反复几百万步，参数逐渐滑到损失最低的地方。这就是梯度下降。

关键结论：梯度是指路的工具。没有梯度，参数就不知道往哪个方向调。

那为什么层数多了，梯度就会衰减？

求第 1 层参数的梯度时，需要链式法则把最后一层的误差逐层传回第一层：

∂L/∂W₁ = ∂L/∂yₙ × ∂yₙ/∂yₙ₋₁ × … × ∂y₂/∂y₁ × ∂y₁/∂W₁

这是 N 个局部导数的连乘。

现在看每个局部导数的值。它们来自两类计算：

  • 激活函数（GELU、Sigmoid 等）的导数 ≤ 1，且输入偏离中心时趋近于 0。比如 Sigmoid 的导数最大只有 0.25；GELU 在负半轴导数也远小于 1
  • 矩阵乘法的导数——权重初始化被控制在接近 1 的范围，不会系统性放大。实践中绝大多数局部导数的绝对值 ≤ 1

于是：一大堆 ≤ 1 的数连乘，乘得越多，结果越接近 0。这不是「深了就会自动消失」，而是指数衰减的数学必然：

每个因子平均 0.5，12 层 → 0.5¹² ≈ 0.00024
每个因子平均 0.5，96 层 → 0.5⁹⁶ ≈ 10⁻²⁹（基本是 0）
每个因子平均 0.9，96 层 → 0.9⁹⁶ ≈ 0.00004

结果：前面几层的梯度几乎为零 → 参数收不到有效的调整信号 → 前面的层卡住不动，模型实际上只有最后几层在学。

🧪 实验 2：梯度衰减模拟器 —— 亲手感受链式法则的威力

模拟梯度从最后一层（右侧）传回第一层（左侧）的过程。每个柱子代表一层的梯度大小。调整层数和衰减因子，观察前面几层的梯度是如何指数级缩水到几乎为零的。然后勾选「残差连接」，看那个 +1 怎么拯救一切。

第 1 层梯度：0.00024
衰减倍数：~4100×

试试 96 层 + 衰减 0.9：梯度降到 10⁻⁴。再勾选残差。

重要澄清：梯度接近 0 ≠ 训练好了

这可能是最容易误解的地方。高中数学里「令导数 = 0，解出极值点」是对的——但那是假设函数是一个光滑的碗（凸函数）。

神经网络的损失函数不是一只碗，而是一片有无数山峰、高原、峡谷的巨大山脉。同一个「梯度接近 0」，可能对应两种完全不同的情况：

真的收敛了 ✅梯度消失了 ❌
损失值已经很低，不再下降还很高，但降不动了
梯度所有层都接近 0前面层接近 0，后面层还有信号
原因真的到了山谷底部信号在路上衰减完了，传不到前面
参数状态稳定，微调即可前面层卡住，根本没学好

实际训练中，判断标准不是梯度的大小，而是损失值（Loss）是否还在下降。训练时观察验证集上的损失曲线——损失连续很多步不再下降就停（这叫 early stopping，早停法）。没人盯着梯度的绝对值说「降到 0.0001 了，训练完成」。损失值是终极裁判，梯度只是指路的向导。关于损失函数的具体计算，将在第 6 课详述。

解决方案：加一条「快捷通道」

残差连接的公式：

output = x + Sublayer(x)

其中 Sublayer(x) 是 Attention 或 FFN 的输出。关键在于 + x 这一项——原始输入 x 被原封不动地加到子层的输出上。

这在反向传播时意味着什么？梯度计算时，对 x 求导：

∂(output)/∂x = 1 + ∂(Sublayer(x))/∂x

那个 1 是关键的——它保证了即使 Sublayer 内部的梯度 ∂(Sublayer)/∂x 衰减到接近零，至少有 1 的梯度能直接传回前面的层。这条不经过任何变换的直通路径，解决了梯度消失问题。

从另一个角度看：有残差连接时，Sublayer 不需要从零开始学习完整的变换。它只需要学「当前输入离理想输出还差多少」——即残差（residual）。如果 Sublayer 什么都没学到（输出全零），output = x + 0 = x，至少不会比原来更差。这让深层网络的训练下限被保证在「至少等于浅层网络」。

§3 Layer Normalization（层归一化）—— 防止数值失控

它解决什么问题

数据每穿过一个子层（Attention 或 FFN），向量里各个维度的数值范围就可能发生漂移。经过多次矩阵乘法和非线性激活后，某些维度的值可能变得极大（几十上百），另一些变得极小（接近零）。如果不加控制，几层之后：

  • 激活函数饱和：GELU 接收极端输入时，输出几乎恒等于 0 或恒等于输入本身，失去了非线性能力
  • 梯度消失或爆炸：极端数值导致反向传播时的梯度也极端

LayerNorm 在每次子层计算之前，把输入强行拉回标准范围。

🧪 实验 3：数值漂移模拟器 —— 为什么需要「拉回来」

模拟一个 20 维向量（柱状图，每根柱子是一个维度的值）连续穿过多个层的效果。每点一次「穿过一层」，数值就会漂移——有些维度暴涨，有些趋近于零。勾选 LayerNorm 后，每层都会把数值重新拉回均值为 0、方差为 1 的范围。

已过层数：0
均值：0.00
标准差：1.00
最大值：2.31

取消勾选 LayerNorm，再点几次「穿过一层」——数值迅速失控。

具体做法

对于一个 token 的 768 维向量，LayerNorm 做三步：

计算均值和方差：求这 768 个数值的平均值 μ 和方差 σ²
标准化：每个值减去 μ，除以 √(σ² + ε)，得到均值为 0、方差为 1 的向量
缩放和平移：乘以 γ（可学习的缩放参数，768 维），加上 β（可学习的平移参数，768 维）。这一步让模型有能力把归一化后的分布调整到最有利于当前层的范围。 
LayerNorm(x) = γ · (x - μ) / √(σ² + ε) + β

为什么 NLP（自然语言处理）用 LayerNorm 而不是更常见的 BatchNorm（批归一化）？因为文本序列长度不一致，同一个 batch（批次）里不同样本的 padding（填充）长度不同，跨 batch 做统计（BatchNorm 的方式）没有意义。LayerNorm 对每个 token 独立计算自己的 μ 和 σ，不受序列长度和 padding 的影响。

组装：一个完整的 Transformer Block

现在把四个组件拼在一起。一个 Block 包含两个子层，每个子层配一个残差连接。LayerNorm 放在每个子层前面（这是现代做法，称为 Pre-LN）：

→ 输出
←── x 跳过 FFN 直接过来 ──

写成伪代码：

def transformer_block(x):
    # 子层 1：Attention
    x = x + attention(layer_norm(x))
    # 子层 2：FFN
    x = x + ffn(layer_norm(x))
    return x

LayerNorm 放前面还是后面？

Post-LN（后置层归一化，原始论文，2017）

x → Attention → + x → LN → FFN → + x → LN

LN 在子层后面。
层数深时训练不稳定，需要学习率 warmup。

Pre-LN（前置层归一化，现代做法，GPT-2 起）

x → LN → Attention → + x → LN → FFN → + x

✅ LN 在子层前面。
训练更稳定，现代 LLM 的标配。

Pre-LN 更好的原因涉及反向传播路径的具体分析，简单说：LN 在子层前面时，梯度回传时不需要穿过 LN 的计算，路径更短，训练更稳定。

叠起来：12 个 Block 串联

一个 Block 做完一轮「交换信息 → 逐位置加工」。但一轮不够。把 12 个 Block 首尾相连：

Embedding 输出
    ↓
[ Block₁ ]  →  第一轮：局部词汇关联
[ Block₂ ]  →  第二轮：短语级别的模式
[ Block₃ ]
   ……
[ Block₁₂ ] →  最后一轮：全局语义、长距离依赖
    ↓
最终输出 → 用于预测下一个 token

每个 Block 有自己的参数（W^Q, W^K, W^V, W^O 用于 Attention；W₁, W₂, b₁, b₂ 用于 FFN；γ, β 用于 LayerNorm）。12 个 Block 互不共享参数。数据从头流到尾，每经过一层就被精炼一次。

最后一层的输出向量，经过一个线性变换 + Softmax，就变成了「下一个 token 在整个词表上的概率分布」。

总结：为什么 Attention 做一次不够
单次 Attention 只能让信息在直接相关的 token 之间流动一次。多层堆叠之后，一个 token 的信息可以逐层扩散到更远的位置——第一层传给邻居，第二层传给邻居的邻居，以此类推。同时，每层的 FFN 在 Attention 聚合的上下文基础上做非线性变换，逐步提取更抽象的模式。

这就是 Transformer Block 存在的根本原因：一轮 Attention + FFN 不够，需要反复做。

🧪 小测验

问题 1/3 — Block 的概念

12 个 Transformer Block 之间共享参数吗？为什么？

点击看答案
不共享。每个 Block 有自己独立的参数。
如果共享参数，12 个 Block 就退化成了同一个 Block 重复计算 12 次——不会增加模型的表达能力。独立参数让每一层可以学到不同抽象级别的模式：浅层学词汇关联，深层学语义逻辑。

问题 2/3 — 残差连接

如果把残差连接去掉（output = Sublayer(x) 而不是 x + Sublayer(x)），堆 96 层 Transformer 会发生什么？

点击看答案
训练不动——前面几层的梯度接近于零，参数无法更新。
没有残差连接时，梯度必须穿过每一层内部的所有矩阵乘法和激活函数才能到达前面的层。96 层下来，梯度呈指数级衰减，前面层的参数几乎得不到更新信号。残差连接的「+ x」提供了梯度绕过子层的直通路径。

问题 3/3 — Attention 和 FFN 的分工

既然 Attention 已经聚合了上下文信息，为什么还需要 FFN？FFN 做的是 Attention 做不到的什么事？

点击看答案
Attention 只能做线性组合（加权平均），FFN 引入了非线性变换。
Attention 的输出是多个 Value 向量的加权和——本质上是线性操作。语言中大量关系是非线性的（否定、条件、类比等），线性组合无法表示。FFN 通过 GELU 激活函数引入非线性，让模型有能力学习这些复杂映射。分工上，Attention 负责「谁和谁相关」，FFN 负责「这个 token 的表示怎么变换」。
