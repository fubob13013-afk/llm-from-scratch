阶段一：预训练 —— 让模型「读尽天下书」

孩子在做什么？

想象一个婴儿，你给它读完人类历史上所有出版过的书、所有网页、所有代码仓库。你给它一个句子前半段，让它猜下一个词。猜错了就调整脑内神经连接；猜对了就强化那条路径。这个「读 → 猜 → 纠错 → 调整」的循环重复几万亿次，它脑内的连接模式逐渐逼近人类语言的统计规律。

数据从哪来？

  • 网页：Common Crawl（一个定期爬取全网的非营利项目），几 PB 级别的原始 HTML
  • 书籍：古腾堡计划、盗版书库（Books3 等，有版权争议）
  • 代码：GitHub 公开仓库，让模型学会编程
  • 维基百科：结构化、高质量的事实性文本
  • 对话：Reddit、论坛帖子，让模型见过"人类怎么聊天"

把这些混在一起，总量通常在几 TB 到几十 TB。GPT-3 的训练数据约 45TB 文本。

🔑 关键认知 预训练是最烧钱的阶段。GPT-4 级别的模型预训练一次花几千万美元电费。这也是为什么只有大公司做预训练——个人和小团队通常在开源预训练模型的基础上往下做。

学到了什么？

预训练结束后的模型叫 Base Model（基座模型）。它是个「什么都读过但不会聊天」的怪胎：你问「法国的首都是哪里？」它不会回答「巴黎」，而是会续写——可能输出「法国的首都是哪里？这是一个常见的地理问题……」因为它见过的训练数据里，问答只是众多文本模式之一，不是唯一的模式。

Base Model 的本质是一个极端强大的文本续写器——你给它什么开头，它就续写什么风格的后续。让它直接对话，等于让一个读了万卷书但没学过社交的人上台演讲：它能说，但不会按照你期望的方式说。

阶段二：微调 —— 教这个「书呆子」聊天

做法

收集几万到几十万条高质量的问答对话，格式类似：

用户：请帮我写一首关于秋天的诗。
助手：秋叶飘落满地金，凉风拂面思故人……

把这些对话喂给 Base Model，继续训练（但学习率调低很多，避免「覆盖」掉预训练学到的知识）。这个阶段叫 SFT（Supervised Fine-Tuning，监督微调）。

结果

模型学会了「对话格式」——知道用户说完该它说，知道要用助手的语气，知道问题对应回答而不是续写。这时候它已经像一个能用的 chatbot 了。

🔑 关键认知 微调不教新知识。模型在预训练阶段已经「知道」巴黎是法国首都——微调只是教会它用「对话」的形式把知识取出来。这就是为什么预训练数据的广度和质量决定模型的上限，微调只是决定它能把上限发挥出多少。

阶段三：对齐 —— 让它「安全且听话」

SFT 之后的模型能聊天了，但有问题：它可能说出冒犯性言论、编造有害信息、或者拒绝回答本应回答的问题。

这个阶段的代表性技术是 RLHF（Reinforcement Learning from Human Feedback，基于人类反馈的强化学习）：

  1. 让模型对同一个问题生成多个回答
  2. 人类标注员给这些回答打分（哪个更好、更安全、更有帮助）
  3. 用这些分数训练一个「奖励模型」，让它学会模拟人类的偏好
  4. 用奖励模型自动给模型的输出打分，强化「好行为」、抑制「坏行为」

对齐之后的模型就是你在 ChatGPT、即梦 AI 对话里实际使用的东西。

阶段四：部署 —— 把模型变成产品

训练好的模型是一个巨大的权重文件（几十到几百 GB），还需要一层工程才能变成你手机/电脑上能用的产品：

  • 量化：把模型参数从 16 位浮点数压缩到 4 位或 8 位，大幅减小体积
  • 推理优化：KV-cache、flash attention 等技术让生成速度变快
  • API 封装：把模型包装成 HTTP 接口，前端发送 prompt 就能拿到回复
  • 前端界面：你看到的聊天窗口、设置面板

对你想做的「离线陪伴机器人」来说，部署阶段最关键——你需要一个量化后的小模型 + 本地推理引擎（如 llama.cpp），才能在手机或树莓派上跑起来。

🤔 那 AI 视频模型（C-Dance 等）呢？

上面讲的是纯文本 LLM的流程。视频生成模型（扩散模型/DiT）的流程类似但更复杂：

维度文本 LLM（GPT 类）视频生成模型（C-Dance 类）
训练数据 纯文本（网页、书籍） 视频 + 文字描述对（几亿组）
学习目标 猜下一个词 从噪声中还原出清晰视频（去噪）
核心架构 Transformer（解码器） DiT（Diffusion + Transformer）
烧钱程度 极高 还要高一个数量级 💀
为什么有的模型好？ 数据质量 + 训练技巧 训练数据的质量和多样性是最核心的差异。C-Dance 可能用了更高质量的数据标注、更多样的动作捕捉、更好的视频-文本对齐

视频模型是一个更大的话题，我们以后可以专门展开。但底层逻辑是一样的：高质量数据 + 大规模训练 + 精巧的架构设计。
