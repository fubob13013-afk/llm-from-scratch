§1 位置编码（Positional Encoding）—— 给每个位置打上独一无二的「指纹」

核心思路

对于位置 pos（第 0 个、第 1 个、第 2 个 token……），生成一个 768 维的位置向量，把它加到词嵌入向量上：

最终输入向量 = 词嵌入向量 + 位置编码向量

这样同一个词在不同位置会有不同的最终表示。「我喜欢猫」里的「我」（pos=0）和「猫喜欢我」里的「我」（pos=2），向量不同。

GPT 用的方法：正弦位置编码（Sinusoidal Positional Encoding）

对于位置 pos 和维度 i（0 ≤ i < 768）：

  • 偶数维度（0, 2, 4, …）：sin(pos / 10000i/768)
  • 奇数维度（1, 3, 5, …）：cos(pos / 10000(i-1)/768)

这个公式的直觉：

  • i 越小（靠前的维度）→ 分母接近 1 → sin/cos 的波长很短 → 相邻位置的编码变化很快 → 捕捉局部位置差异
  • i 越大（靠后的维度）→ 分母很大 → sin/cos 的波长很长 → 编码变化缓慢 → 捕捉远距离位置关系

不同频率的 sin/cos 波叠加在一起，让每个位置拥有一个独一无二的向量指纹。而且相邻位置的指纹很相似（这对模型学习局部模式有帮助），远距离位置的指纹差异很大。

为什么用 sin/cos 而不是让模型自己学？

两种方案：

方案做法优点缺点
可学习位置编码 给每个位置随机初始化一个向量，训练时让模型自己调整 灵活，GPT-2 及之后很多模型用这个 只能处理训练时见过的位置长度，无法泛化到更长的序列
正弦位置编码 用 sin/cos 公式计算，不参与训练 可以泛化到任意长度；相邻位置天然相似 固定规则，不如可学习的灵活

原始 Transformer 论文和 GPT-1 用正弦编码。GPT-2 及之后的模型大多改用可学习位置编码。但核心思想完全一样：给每个位置一个独一无二的向量，加到词嵌入上。

§2 完整 GPT 架构 —— 拼好最后一块拼图

现在可以画出完整的前向传播流程图了：

输入 Token ID 序列：[15, 27, 8, 42, …, 91]   (1024 个 token)
              │
              ▼
    ┌─────────────────────┐
    │  Token Embedding    │  查表：每个 ID → 768 维语义向量
    │  Position Encoding  │  + 768 维位置指纹向量
    └─────────────────────┘
              │
              ▼
    ┌─────────────────────┐
    │     Dropout         │  随机关掉 10% 的神经元，防止过拟合
    └─────────────────────┘
              │
              ▼
    ╔═════════════════════╗
    ║  Transformer        ║
    ║  Block 1            ║  LN → Attention → +x → LN → FFN → +x
    ║  Block 2            ║  参数不同，结构相同
    ║  …                 ║
    ║  Block 12           ║
    ╚═════════════════════╝
              │
              ▼
    ┌─────────────────────┐
    │  Final LayerNorm    │  最后一次归一化
    └─────────────────────┘
              │
              ▼
    ┌─────────────────────┐
    │  Linear + Softmax   │  768 维 → 词表大小（50000 维）
    │                     │  Softmax 转成概率分布
    └─────────────────────┘
              │
              ▼
    输出：每个位置上，下一个 token 的概率分布
    [P(北京|…), P(上海|…), P(的|…), …]  (50000 个概率)

写成伪代码

def gpt_forward(token_ids):
    # 1. 嵌入
    x = token_embedding(token_ids)      # [batch, 1024, 768]
    x = x + position_encoding(x)        # + 位置指纹

    # 2. 过 12 层 Transformer Block
    for block in transformer_blocks:    # 12 个 Block，参数各自独立
        x = block(x)

    # 3. 输出
    x = layer_norm_final(x)
    logits = linear(x)                  # [batch, 1024, 50000]
    probs = softmax(logits)             # 转成概率

    return probs  # 每个 token 位置预测下一个 token

这就是 GPT-2 小型版（124M 参数）的完整计算流程。大模型（GPT-3、GPT-4）的区别只在：更多层（96 层）、更宽（12288 维）、更多数据——结构完全一样。

§3 文本生成 —— 一个 token 一个 token 地产出

自回归解码（Autoregressive Decoding）

模型一次只能预测一个 token。要生成一整段文本，需要把刚才生成的 token 接在输入后面，再喂给模型预测下一个。如此循环：

第 1 步：输入「中国的首都是」→ 模型预测「北」
第 2 步：输入「中国的首都是北」→ 模型预测「京」
第 3 步：输入「中国的首都是北京」→ 模型预测「。」
第 4 步：输入「中国的首都是北京。」→ 模型预测 <EOS>（结束符）
                                    → 生成完成

每次预测时，模型输出的是整个词表（50000 个 token）上的概率分布。怎么从概率分布中选出一个 token？这有好几种策略。

三种采样策略

策略做法效果
贪心（Greedy） 每次都选概率最高的 token 确定性强，但容易陷入重复——模型会一直选最「安全」的词
Temperature 采样 把概率除以 temperature，再按比例随机选 T=0 → 接近贪心；T=1 → 自然随机；T→∞ → 完全随机。控制「创造力」
Top-k / Top-p 只从概率最高的 k 个中选（top-k），或只从累积概率达到 p 的最小集合中选（top-p） 过滤掉低概率的「噪声」选项，在质量和多样性间取平衡

Temperature 在做什么？

模型先算出每个 token 的原始分数（logits），用 softmax 转成概率。Temperature 在 softmax 之前把 logits 除以 T：

概率 = softmax( logits / T )
  • T < 1（如 0.5）：logits 变大 → softmax 后高分 token 概率更高，低分 token 概率更低 → 输出更「确定」
  • T = 1：不改动，按原始概率采样
  • T > 1（如 1.5）：logits 缩小 → softmax 后概率分布更均匀 → 输出更「随机」

把一切串起来

从第 1 课到第 7 课，你走完了 GPT 的完整旅程：

  1. 数据准备（第 2 课）：原始文本 → 分词 → Token ID 序列 → 滑动窗口切分
  2. 词嵌入（第 3 课）：每个 Token ID → 768 维语义向量 + 位置向量

  5. 训练循环（第 6 课）：前向 → 损失 → 反向 → 更新，反复百万次
