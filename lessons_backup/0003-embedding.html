🧭 第3课 · 词嵌入（Embedding）

上节课我们把文字变成了 Token ID（整数）。但这串整数怎么变成模型能「理解」的东西？答案：给每个词一个坐标。

⚠️ 本课前提 你需要知道什么是 Token ID——上节课讲过。简单回顾：经过分词和词表映射后，「我喜欢人工智能」变成了类似 [15, 42, 38, 89, 91] 这样的整数序列。每个整数就是一个 Token ID。

问题：Token ID 有什么致命缺陷？

假设词表里：

"国王"  → Token ID = 15
"王后"  → Token ID = 382
"苹果"  → Token ID = 16

从数字上看，「国王」（15）和「苹果」（16）只差 1——比「国王」和「王后」（差 367）「更接近」。但语义上显然「国王」和「王后」更相关。

Token ID 只是一个编号，它不携带任何含义。15 和 16 的差距是人为分配的，不代表这两个词有任何关系。

计算机需要的是：把每个词表示成一个能体现语义的「位置」——含义相近的词，位置也相近。

什么是向量？—— 30 秒速通

你可能听过「向量」这个词但觉得抽象。其实它就是一组数字，用来描述一个点在空间中的位置。

🔑 颜色 RGB 类比（你肯定懂这个）
你在 PS 里调过色，知道每个颜色都可以用 (R, G, B) 三个数字表示：

红色
(230, 57, 70)
橙红色
(230, 115, 70)
蓝色
(69, 123, 157)

注意：红色和橙红色的 RGB 值很接近（只有 G 值差了一点），而它们和蓝色的值差得很远。这恰好就是 Embedding 要做的事——相似的东西，坐标也相似。

一个 RGB 颜色是一个 3 维向量。词嵌入（Word Embedding）做的事完全一样，只是维度更多：不是 3 维，而是 768 维、2048 维甚至 4096 维。

为什么需要这么多维？因为颜色的差异只需要 3 个维度就能描述（红绿蓝），但人类语言的语义差异极其复杂——性别、时态、感情色彩、正式程度、领域……需要几百上千个维度来刻画。

2 为什么这能行？—— 分布式语义假说

语言学里有一个著名的观察（不是假说，是大量实验验证过的规律）：

「一个词的含义，由它身边的词决定。」
— John Rupert Firth, 1957

什么意思？看这两个句子：

我喝了一杯___。    ← 空格里大概率是「水」「茶」「咖啡」「牛奶」……
我吃了一个___。    ← 空格里大概率是「苹果」「面包」「鸡蛋」……

「水」「茶」「咖啡」经常出现在相同的上下文里（「喝」的后面），所以它们的嵌入向量会很接近。「苹果」「水」虽然都可以出现在空格里，但一个搭配「吃」一个搭配「喝」——它们的向量就会远一些。

模型在训练时，并不是被显式告知「国王和王后有关」。它只是在几万亿次「看到前面猜后面」的训练中，发现了：出现在相似上下文里的词，调整它们的向量让它们靠近，能降低预测误差。

768 维意味着什么？每个词不是二维平面上的一个点，而是在一个 768 维的超空间中的一个点。这个空间有 768 个互相垂直的方向，每个方向编码一种语义特征——某几个维度可能对应「性别」，某几个对应「大小」，某几个对应「正式程度」……虽然实际上每个维度并不一定对应单一的人类可解释的特征，但大致是这个逻辑。

4 位置也很重要 —— 位置嵌入（Preview）

还有一个问题：「国王吃了苹果」和「苹果吃了国王」——词完全一样，顺序不同，意思天差地别。

但 Embedding 只给每个词一个固定坐标——「国王」不管出现在句子第 1 位还是第 5 位，查出来的向量都是一样的。模型怎么知道词序？

答案是位置嵌入（Position Embedding）：在词嵌入的基础上，再叠加一个只和位置有关的向量。

最终输入向量 = 词嵌入（"苹果"的语义向量）+ 位置嵌入（"第 4 个位置"的位置向量）

这样同一个词在不同位置会有不同的最终表示。位置嵌入的具体算法（正弦位置编码、可学习位置编码等）我们到 Transformer 那课再详细展开。现在你只需要知道：Embedding 有两层——语义的 + 位置的。

🧪 小测验

问题 1/3 — 查表

GPT-2 的 Embedding 表格有多少行？多少列？（给出数量级即可）

点击看答案
约 50,000 行 × 768 列。
行数 = 词表大小（GPT-2 词表约 50,257 个 token），
列数 = 嵌入维度（GPT-2 用 768）。
总数约 3800 万个数字——这些都是模型要学的参数。

问题 2/3 — 语义距离

训练好的 Embedding 中，「猫」和「狗」的向量距离近，还是「猫」和「汽车」的向量距离近？为什么？

点击看答案
「猫」和「狗」更近。
因为在训练数据中，「猫」和「狗」频繁出现在相似的上下文里（"我养了一只___"、"___在沙发上睡觉"等），模型在训练中把它们的向量调到了彼此附近。「汽车」出现在完全不同的上下文里，向量自然离得远。

问题 3/3 — 位置的作用

「狗追猫」和「猫追狗」——分词后 token 完全相同，只是顺序不同。只用词嵌入（不加位置嵌入）的话，模型能看到区别吗？

点击看答案
不能。只用词嵌入的话，「狗」无论在第 1 位还是第 3 位，查出来的向量一模一样。两个句子送入 Transformer 的矩阵完全相同，模型无法区分谁追谁。

这就是为什么必须加位置嵌入——同一个词在不同位置，最终输入向量不同。
