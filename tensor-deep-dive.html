📦 张量完全理解指南

如果你看完第8课对「张量」还是一头雾水——这篇文章就是为你写的。
我们从最基础的直觉开始，用图形化+类比的方式，一步步把张量讲透。

一、一句话理解张量

张量 = 一个装了数字的盒子，盒子的「形状（shape）」告诉你每个方向上有几个数字。

把张量想象成一个快递盒：

📦
张量 = 盒子
一个容器，里面装着很多数字
📐
Shape = 尺寸
盒子的长宽高，告诉你数字怎么排列
⚡
GPU = 叉车
知道盒子的尺寸后，叉车可以一次性搬走整个盒子，而不是一个一个捡

关键点：为什么「给数据加上形状」就能一次性处理整个数组？因为 GPU 是一块专门做并行计算的芯片。当你告诉它「这个张量的 shape 是 (3, 4)」，GPU 就知道这 12 个数字的排列方式，可以同时对它们做运算——而不是像 CPU 那样一个接一个地处理。这就是为什么深度学习必须用张量：速度差 100 倍以上。

二、从 0 维到 5 维：一步步可视化

张量最让人困惑的地方在于「维度」这个词。下面我们用同一种视觉语言，从最简单的标量开始，一步步升级到 5 维张量。你会发现每一级只是在上一级外面多加一层括号而已。

0 维 —— 标量（Scalar）

就是一个单独的数字。没有方向，没有行列，就一个数。

42

Python: 42 ｜ Shape: () ｜ 也叫「0 维张量」

❓ 为什么 shape 是 () 或 (1,)？(1,) 看起来跟向量 (3,) 一样啊？

好问题！这是张量学习中最容易混淆的地方。

() 是 PyTorch 里真正的标量——0 个维度，就是一个数。比如 torch.tensor(42) 的 shape 就是 torch.Size([])（显示为空括号）。

(1,) 是一个只含 1 个元素的向量——它和 (3,) 一样都是1 维张量，只不过长度是 1。就像「只住了 1 个人的 1 号房间」vs「住了 3 个人的 3 号房间」——它们都是房间（1 维），只是人数不同。

用代码验证：torch.tensor([42]).shape → torch.Size([1])，而 torch.tensor(42).shape → torch.Size([])。注意区别：方括号 [42] 创建了一个列表（1 维），裸数字 42 才是标量（0 维）。

1 维 —— 向量（Vector）

把几个标量排成一条线，就有了向量。注意：向量的「维」指的是这条线上有几个数，不是「几维空间」。一个 3 维向量就是 3 个数排成一排。

1
2
3

Python: [1, 2, 3] ｜ Shape: (3,) ｜ 1 维，长度 3

❓ Python 里为什么写成 [1, 2, 3]，而不是 (3,)？

因为它们是两个完全不同的东西：

[1, 2, 3]（方括号）是 Python 的 list（列表）——它写的是里面装的值本身。1、2、3 就是这三个数。

(3,)（圆括号）是 shape（形状）——它是张量的「尺寸描述」，不是你往里面填的值。shape 是容器的大小，list 是容器的内容物。

打个比方：你买了一个鞋盒，上面写着「尺码 42」。[鞋, 鞋垫, 鞋带] 是盒子里的东西，(3,) 是告诉你「这个盒子能装 3 样东西」。不要把标签和内容搞混了。

2 维 —— 矩阵（Matrix）

把多个向量纵向叠起来，就变成了矩阵——有行和列两个方向。shape (3, 4) 读作「3 行，4 列」。

3 维 —— 3D 张量

把多个矩阵叠成一摞，就变成了 3 维张量。shape (2, 3, 4) 读作「2 个矩阵，每个 3 行 4 列」。

4 维 —— 批次维度登场

把多个 3D 张量再叠一层，就是 4 维。这听起来抽象，但在深度学习中极其常见：

🎨 例：一批图片的 shape = (B, H, W, C)

B=批次 几张图片一起处理
H=高度 每张图片多少像素高
W=宽度 每张图片多少像素宽
C=通道 RGB 3 个颜色通道

比如 (32, 224, 224, 3) = 32 张 224×224 的彩色图片，一次丢进模型。
这就是 GPU 并行的威力：32 张图同时处理，而不是一张一张来。

5 维及以上 —— 可以一直叠下去

❓ 有没有 shape 是 (B, L, D, C, A) 的张量？它叫「五维张量」吗？

是的，它就是一个 5 维张量。N 维张量这个称呼只看 shape 里有几个数字，不看这些字母代表什么含义。shape 里出现 5 个数字 = 5 维。

但实际中几乎不会出现你举的这个 (B, L, D, C, A) 组合——每个领域有自己的命名习惯。NLP 领域最常见的 5 维张量长这样：

(batch, num_heads, seq_len, seq_len, d_head)

这是多头注意力机制里的张量：把 Q、K、V 拆成多个「头」，每个头独立计算注意力。比如 (2, 12, 128, 128, 64) = 2 个句子，12 个注意力头，128×128 的注意力矩阵，每个头 64 维。

规律：每多一个维度，就是在外层多加了一组方括号，把原来的一整坨再打包一次。理论上可以叠到任意高维，GPU 都能处理。

四、LLM 里最常见的形状：(batch, seq_len, d_model)

这是第 8 课提到的一句话，但没有展开。现在你已经理解了 shape 的含义，我们把它拆开揉碎。

先看一个具体的例子

假设你想让 GPT 同时处理 2 句话：

批次中的 2 个句子

句子 1：我 爱 北京
我爱北京
句子 2：天安 门 上 太阳 升
天安门上太阳升

问题：句子长度不一样！句子 1 只有 3 个 token，句子 2 有 5 个 token。
解决方案：填充（padding）——把短的句子补到一样长。

句子 1（补齐后）
我爱北京⬚⬚
句子 2（补齐后）
天安门上太阳升

三个维度的含义

维度字母含义本例中的值
第 1 维 batch 批次大小：同时处理几句话 2（2 个句子）
第 2 维 seq_len 序列长度：每句话最多几个 token 5（统一补到 5 个 token）
第 3 维 d_model 向量维度：每个 token 用多少个数字表示 768（GPT-2 的维度）

🎨 可视化：(2, 5, 768) 张量的结构

batch=0 第 1 个句子（"我爱北京"）
seq=0 "我" → [768 个数字的向量]
seq=1 "爱" → [768 个数字的向量]
seq=2 "北京" → [768 个数字的向量]
seq=3 "⬚" → [768 个数字的向量（padding）]
seq=4 "⬚" → [768 个数字的向量（padding）]
batch=1 第 2 个句子（"天安门上看太阳升"）
seq=0 "天安" → [768 个数字的向量]
seq=1 "门" → [768 个数字的向量]
seq=2 "上" → [768 个数字的向量]
seq=3 "太阳" → [768 个数字的向量]
seq=4 "升" → [768 个数字的向量]

总元素数 = 2 × 5 × 768 = 7,680 个数字。GPU 一次性处理这 7,680 个数字。

为什么需要 batch 维度？

🐢
没有 batch
一次只处理 1 句话
GPU 大部分时间在等待
训练一个模型要几个月
🐇
有 batch
一次处理 32-64 句话
GPU 满负荷运转
训练时间缩短 30+ 倍
🧠 记住这个公式
张量 shape = (batch, seq_len, d_model) 是整个 Transformer 架构中最核心的数据形状。
从 Embedding 查表 → 注意力计算 → FFN → 输出，数据始终保持着这个三维结构在模型里流动。

核心要点清单

  1. 张量 = 有形状的数字容器，标量/向量/矩阵都是张量的特例
  2. Shape = 从外到内数括号的层数，(2,3,4) 读作「2 个矩阵，各 3 行 4 列」
  3. Python list [1,2,3] 是值本身，shape (3,) 是对结构的描述——两回事
  4. 向量「3 维」指有 3 个数，不是 3 维空间——别把值的维度和空间的维度搞混
  5. (batch, seq_len, d_model) 是 LLM 中最核心的三维形状，贯穿整个 Transformer
  6. 元素总数 = shape 所有数字相乘，这条规则永远成立
  7. GPU 知道 shape 就能并行——就像叉车知道箱子尺寸就能一次搬走

六、回到第 8 课

现在你应该能看懂第 8 课里的这一段了： 

input_ids = torch.tensor([[1, 5, 8, 2]])  # 1 个句子，4 个 token
print(f"输入 shape: {input_ids.shape}")     # (1, 4)

x = embed(input_ids)                        # 查 Embedding 表
print(f"Embedding 后 shape: {x.shape}")     # (1, 4, 64)
                                            # ↑ batch=1, seq_len=4, d_model=64

out = ffn(x)                                # 过 FFN
print(f"FFN 后 shape: {out.shape}")         # (1, 4, 256)
                                            # ↑ batch=1, seq_len=4, d_model=256

每一步 shape 的变化都是有意义的：(1, 4) → (1, 4, 64) 是每个 token 从「一个 ID 数字」变成了「64 个特征的向量」；(1, 4, 64) → (1, 4, 256) 是 FFN 把向量扩到了 256 维。理解了 shape 的变化，你就理解了一半的 Transformer。
