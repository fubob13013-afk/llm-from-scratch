🚀 Phase 5 · 进阶方向总览

恭喜！你已经完成了从「零基础」到「能写 GPT 代码 + 能训练 + 能评估」的完整旅程。 Phase 5 的四个方向不是必学——按你的兴趣和目标任选 1~2 个深入。

方向 A：微调与对齐

SFTRLHFDPOLoRA

解决什么问题？ Phase 1-4 训练的是 Base Model（基础模型）——它只会「续写文本」，不会「回答问题」。 ChatGPT 的回答能力来自微调和对齐。

A1 · SFT（监督微调）

用几千条「人类写的优质对话」继续训练模型。Base Model 学会「用户问→我答」的格式。 数据量不大（几千到几万条），但质量要求高。

A2 · RLHF / DPO（对齐人类偏好）

SFT 后模型会回答问题了，但回答可能不安全、不有帮助、不符合价值观。 RLHF（从人类反馈中强化学习）让人类标注「哪个回答更好」来进一步调整模型。 DPO 是 RLHF 的简化版——不需要训练奖励模型，直接优化，近年来更流行。

A3 · LoRA / QLoRA（低秩适配）

不想更新全部 124M 参数？LoRA 在原始权重旁边加小矩阵只训练这小部分—— 参数减少 99% 以上，效果接近全参数微调。QLoRA 更进一步：用 4-bit 量化 + LoRA，7B 模型在单个消费级 GPU（24GB）上就能微调。

🎯 为什么 Bob 可能需要这个方向？ 如果你想做一个「离线陪伴机器人」——一个能聊天、有性格的 AI——这个方向是必经之路。 Base Model（L12 的 GPT-2）只会续写，需要 SFT 才能对话。

方向 B：量化与部署

INT8INT4llama.cppOllama

解决什么问题？ 124M 的模型文件（FP32 格式）约 500MB——但 7B 的模型就是 28GB。 量化把 32 位浮点数压成 8 位甚至 4 位整数，模型缩小 4-8 倍，推理加速 2-4 倍。

B1 · 量化原理

核心思想：神经网络参数不需要 32 位的精度。把 FP32 → INT8（8 位），精度损失很小。 如果进一步量化到 INT4（4 位），每个参数只占半个 byte——7B 模型从 28GB → ~3.5GB。

B2 · llama.cpp / Ollama

在普通电脑（甚至树莓派）上跑 LLM 的终极工具。llama.cpp 用 C++ 优化推理，支持 CPU-only 运行。 Ollama 是 llama.cpp 的桌面版——安装后一行命令就能在本地跑各种开源模型。

🎯 为什么 Bob 可能需要这个方向？ 如果你想做一个能本地运行、不依赖云端的 AI 助手——这个方向教你做到。 7B 模型量化后可以在 16GB 内存的笔记本上运行。

方向 C：多模态

CLIPDiffusionDiT语音

解决什么问题？ 让模型不仅能读文字，还能看懂图片、生成视频、听懂语音。

C1 · CLIP — 图文对齐

训练一个模型把「图片」和「描述图片的文字」映射到同一个向量空间。 之后就可以用文字搜图片、用图片搜文字——这是所有多模态模型的基础。

C2 · Diffusion + DiT — 图像/视频生成

Stable Diffusion（文生图）和 Sora/即梦（文生视频）的核心技术。 与 LLM 的 Transformer 架构同根同源——DiT（Diffusion Transformer）就是把 Transformer Block 用在图像生成上。

C3 · 语音对话

ASR（语音→文字）+ LLM + TTS（文字→语音）= 一个能说能听的 AI 助手。

🎯 为什么 Bob 可能需要这个方向？ Bob 在用即梦做 AI 视频创作。理解 Diffusion/DiT 的底层原理，能让你更好地控制视频生成。

方向 D：Agent 与工具使用

Function CallingRAGMemory

解决什么问题？ 基础 LLM 只知道训练数据里的内容，不会查资料、不会用工具。 Agent 给模型装上「手脚」——能搜索网页、调用 API、读写文件。

D1 · Function Calling（工具调用）

让模型输出结构化指令（如 {"tool": "search", "query": "今天北京天气"}）， 外部程序执行后把结果还给模型继续回答。

D2 · RAG（检索增强生成）

模型回答前先去知识库检索相关文档，把文档内容作为上下文拼进 prompt。 解决幻觉问题的核心手段——fact 来自检索结果，不靠模型记忆。

D3 · Memory（记忆系统）

让模型在跨多轮对话中记住用户的重要信息。技术路线：向量数据库存储对话摘要 → 检索相关记忆 → 嵌入 prompt。

🎯 为什么 Bob 可能需要这个方向？ 「离线陪伴机器人」需要：记住你的偏好（Memory）+ 能查天气/新闻（Function Calling）+ 回答基于事实（RAG）。 这三个加起来 = 一个能真正帮到你的 AI 助手。

推荐学习路径

根据 Bob 的「离线陪伴机器人」目标，推荐优先级：

  1. 方向 A（微调） — 让模型从「续写」变成「对话」，这是起点
  2. 方向 D（Agent） — 加上 RAG 和 Memory，让对话有事实依据和个人记忆
  3. 方向 B（量化部署） — 把模型压缩到能在自己电脑上跑
  4. 方向 C（多模态） — 加上视觉/语音能力（这个最复杂，放最后）
