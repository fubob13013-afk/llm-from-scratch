概念一：nn.Embedding —— 把「查表」写成代码

回顾 L3：Embedding 到底是什么？

假设词表有 50257 个 token，每个 token 用一个 768 维的向量表示。 整个 Embedding 层就是一张 50257 × 768 的大表格：

  • 每一行是一个 token 的「语义向量」
  • 输入一个 token ID → 取出对应的那一行 → 得到 768 个数字
  • 这些数字一开始是随机初始化的，在训练中逐渐学会语义（「猫」和「狗」的向量越来越近）

在 PyTorch 里，这张大表就是一行代码：

embed = nn.Embedding(50257, 768)创建一张 50257 行 × 768 列的大表
# embed.weight shape → (50257, 768).weight 就是这张表的所有数字，50257 行，每行 768 个

查表操作也很简单——把 token ID 张量喂进去就行：

token_ids = torch.tensor([[42, 318, 7]])一个 batch，1 句话，3 个 token
x = embed(token_ids)查表：取第 42、318、7 行
# x.shape → (1, 3, 768)1 句话 × 3 个词 × 每个 768 维
🔗 回到 L2 这些 token ID（42, 318, 7）不是凭空来的——它们是 L2 讲的 BPE 分词器的输出。 原始文本 → BPE 分词 → Token ID → 这里 → Embedding 向量。

概念二：正弦位置编码 —— 从公式到代码

为什么需要位置编码？

回到 L3 的核心洞察：Embedding 只管「什么词」，不管「在哪个位置」。 「我喜欢猫」和「猫喜欢我」——三个词的 Embedding 向量完全一样，但意思完全不同。

位置编码就是给每个位置的向量加上一个独一无二的「位置指纹」， 让模型能区分「第 1 个词」和「第 3 个词」。

正弦位置编码的公式

GPT 用的不是唯一的方式，但正弦位置编码是最经典、最优雅的方式。 它用 sin 和 cos 函数生成位置编码，不需要训练，直接算：

PE(pos, 2i)      = sin( pos / 100002i/dmodel )
PE(pos, 2i+1) = cos( pos / 100002i/dmodel )

这个公式看起来吓人，拆开来就很简单：

  • pos：位置编号（0, 1, 2, 3, …）——这是第几个词
  • i：维度对的编号（0, 1, 2, …）——向量的第几个「位置对」
  • dmodel：向量的总维度数（比如 768）
  • 关键设计：偶数维度（2i）用 sin，奇数维度（2i+1）用 cos
  • 100002i/d：控制波浪频率——越靠前的维度对，波浪越快；越靠后，波浪越慢

🔰 零基础详解：正弦位置编码到底在干什么

上面的解释如果你还觉得抽象，没关系——这一节用纯大白话 + 具体比喻，把公式的每一部分拆开揉碎。 不需要你懂高数，只需要小学水平的加减乘除和对 sin/cos 的最浅印象。

🎹 核心比喻：钢琴卷帘窗

如果你用过任何音频编辑软件（达芬奇的 Fairlight、Audition、Logic Pro 都行），一定见过钢琴卷帘窗（Piano Roll）：

  • 横轴 = 时间（第 1 拍、第 2 拍、第 3 拍……）
  • 纵轴 = 音高（上面是高音短笛，下面是低音贝斯）
  • 每个格子 = 某一拍、某一个音高上，有没有音符

正弦位置编码做的事，几乎一模一样。想象一张 768 行的大表格：

  • 横轴（列）= 句子的第几个词——第 0 列 = 第 1 个词，第 3 列 = 第 4 个词
  • 纵轴（行）= 向量的第几个维度——第 0 行是维度 0，第 767 行是维度 767
  • 每个格子里填一个介于 -1 到 +1 之间的数字

但和钢琴卷帘窗不同的是：每个格子的数字不是随便填的，而是由 sin 和 cos 函数生成的波浪图案。 前几行（小维度编号）是高频短波——每个位置数值剧烈跳动； 后几行（大维度编号）是低频长波——走完整句话才完成半个起伏。

这样，每个词位置拿到的 768 个数字组合，在这个「波浪系统」里都是独一无二的。 就像每个人的指纹由不同密度的纹路混合而成——细纹 + 粗纹 + 螺旋 = 唯一身份。

📐 公式逐字翻译

先再看一眼公式，然后我们用颜色标注逐块翻译：

PE(pos, 2i)   = sin( pos / 100002i/dmodel )
PE(pos, 2i+1) = cos( pos / 100002i/dmodel )
符号大白话举个例子
PE 「位置编码」这个函数的缩写，就像 f(x) 里的 f。PE(位置, 维度) = 这个格子填什么数字 PE(3, 0) = 「第 4 个词的第 0 维填什么」
pos 位置编号，从 0 开始数：第 1 个词 pos=0，第 2 个词 pos=1…… 句子「我 喜欢 猫」→ 三个词的 pos 分别是 0, 1, 2
i 维度对的编号。768 维分成 384 对，i 从 0 到 383。i=0 是第 0 对（维度 0+1），i=383 是第 383 对（维度 766+767） i=0 → 管维度 0 和 1；i=5 → 管维度 10 和 11
2i 第 i 对里的偶数维度编号 i=0 时 2i=0（维度 0）；i=5 时 2i=10（维度 10）
2i+1 第 i 对里的奇数维度编号 i=0 时 2i+1=1（维度 1）；i=5 时 2i+1=11（维度 11）
dmodel 向量的总维度数，一个固定常数（GPT-2 里 = 768） 全文不变，始终是 768
10000 频率缩放基数，一个经验常数。用它来确保最高频和最低频覆盖的范围刚好适合模型的序列长度 不是推导出来的，是试出来的好数字（详见下文）

🧠 核心直觉：pos 和 i 的区别——一句话记住

很多初学者卡在「pos 和 i 到底谁是谁」上。记住这个画面：

pos 是「第几个词」→ 竖向看，遍历整句话的每个词
i 是「第几对维度」→ 横向看，遍历每个词向量内部的维度对

具体来说：当你固定 pos=3（只看第 4 个词），让 i 从 0 跑到 383，你就得到了这个词的完整 768 维位置编码。 当你固定 i=0（只看最前面那对维度），让 pos 从 0 跑到句子末尾，你就看到了这一对维度在所有位置上的波浪变化。

📏 为什么是 10000？——「唱片」比喻

10000 这个数字看起来很奇怪，但它只做一件事：控制波浪频率的范围。

想象一张黑胶唱片：

  • 唱片的外圈纹路密集（高频），内圈纹路稀疏（低频）
  • 从外圈到内圈，频率连续渐变
  • 10000 就是决定「外圈多密、内圈多疏」的那个旋钮

具体计算一下你就懂了：

dmodel = 768

// 最高频维度对（i=0）：
分母 = 100000/768 = 100000 = 1
→ sin(pos / 1) → 每 ≈6 个位置完成一次上下震荡 → 超高频 🔔

// 中间维度对（i=191）：
分母 = 10000382/768 = 100000.5 = 100
→ sin(pos / 100) → 每 ≈628 个位置完成一次震荡 → 中频 🎻

// 最低频维度对（i=383）：
分母 = 10000766/768 ≈ 10000
→ sin(pos / 10000) → 在 1024 个位置范围内几乎是一条缓坡 → 超低频 🎸

为什么恰好是 10000？因为这个值能保证：最高频精确到能区分相邻词，最低频平缓到能给整句话一个「区域感」，中间覆盖了全部过渡频率。 它不是数学推导的必然，是论文作者试出来效果好的工程经验值。换成 5000 或 20000 也能跑，但 10000 在当时（序列长度 ≈ 512~1024）效果最好。

🌊 为什么必须 sin 和 cos 成对出现？——「钟表指针」比喻

注意公式里每对维度（2i 和 2i+1）用的是同一个分母（同一个频率），只是一个填 sin、另一个填 cos。 这不是随便选的——它背后藏着一个精巧的设计。

想象一个钟表的指针：

  • 要描述指针在表盘上的位置，你需要两个数字：水平偏移（cos）和垂直偏移（sin）
  • 光知道 sin（垂直偏移）不够——指针在 12 点和 6 点都是 sin=0，你分不清
  • 但 sin + cos 组合可以唯一确定指针指向几点钟

每一对维度（sin + cos）就像是以不同速度旋转的钟表指针：

  • i=0（高频对）：指针转得飞快——每走过 6 个词就转完一圈
  • i=383（低频对）：指针转得极慢——走完整篇文章才转几十度
  • 所有 384 对维度合在一起 = 384 个不同转速的钟表指针同时记录位置

这带来了一个关键好处：相对位置可计算。利用三角恒等式 sin(a+b) = sin(a)cos(b) + cos(a)sin(b)， 模型可以通过线性变换从位置 A 的编码算出位置 B 的编码。 这意味着模型学到的不是「位置 7 长这样」的死记硬背，而是「这两个词相距多远」的相对关系。

🎯 完整走一遍：三个词是怎么被「编码」的

假设句子只有三个词：「我 喜欢 猫」，dmodel=4（简化到 4 维），我们来手动走一遍：

步骤操作结果
1 三个词的 token ID：[42, 318, 7] 每个 ID 是一个整数，没有位置信息
2 Embedding 查表：取出每行 4 个数字 词 0「我」：  [0.21, -0.45,  0.83, -0.12]
词 1「喜欢」：[-0.67,  0.34, -0.21,  0.75]
词 2「猫」：  [0.92, -0.18,  0.44, -0.56]
3 位置编码计算（sin/cos 公式） pos=0：  [sin(0), cos(0), sin(0), cos(0)] = [0, 1, 0, 1]
pos=1：  [sin(1), cos(1), sin(1/100), cos(1/100)] ≈ [0.84, 0.54, 0.01, 1.00]
pos=2：  [sin(2), cos(2), sin(2/100), cos(2/100)] ≈ [0.91, -0.42, 0.02, 1.00]
4 逐元素相加（词嵌入 + 位置编码） 词 0：  [0.21+0,   -0.45+1,    0.83+0,   -0.12+1] = [0.21, 0.55, 0.83, 0.88]
词 1：  [-0.67+0.84, 0.34+0.54, -0.21+0.01, 0.75+1.00] = [0.17, 0.88, -0.20, 1.75]
词 2：  [0.92+0.91, -0.18-0.42, 0.44+0.02, -0.56+1.00] = [1.83, -0.60, 0.46, 0.44]

观察第 3 步中的位置编码：维度 0-1（i=0，高频，分母=1）数值在三个位置上剧烈变化（0→0.84→0.91）； 维度 2-3（i=1，低频，分母=100）数值变化极小（0.01→0.02），几乎不变。

加到词嵌入上之后（第 4 步），同一个词「我」在 pos=0 和 pos=2 的最终向量完全不同—— 尽管词嵌入一样，位置编码的加入让模型可以区分「开头位置的我」和「中间位置的我」。

✅ 一句话总结

正弦位置编码 = 给每个位置发一张由 384 种不同转速的 sin/cos 波浪叠加而成的「指纹」。

高频波（前列维度）提供精确位置 → 区分「第 3 个词」和「第 4 个词」
低频波（后列维度）提供区域位置 → 判断「在句子前半段还是后半段」
全部叠加在一起 → 每个位置的 768 个数字组合在整个序列中独一无二

这张「指纹」加到词嵌入向量上，Token ID 就从「什么词」变成了「什么词 + 在哪个位置」。
💡 为什么用 sin/cos 而不是随机数？
两个好处：
1. 值域稳定：sin/cos 输出永远在 [-1, 1] 之间，不会像可学习参数那样出现极端值
2. 相对位置可外推：sin(a+b) = sin(a)cos(b) + cos(a)sin(b)——模型可以从中学到两个位置之间的相对距离，而不只是绝对位置。这意味着训练时只见过 1024 长度的序列，推理时可能处理更长的序列

把公式变成代码

现在把上面的公式逐行翻译成 Python（PyTorch 版）：

def sinusoidal_position_encoding(seq_len, d_model):输入：序列长度（几个词）和向量维度
pe = torch.zeros(seq_len, d_model)先创建一张全 0 的「空表」，shape=(seq_len, d_model)
position = torch.arange(seq_len).unsqueeze(1)位置编号：[[0],[1],[2],…] shape=(seq_len, 1)
div_term = torch.arange(0, d_model, 2)生成偶数索引：[0, 2, 4, 6, …] 对应公式里的 2i
div_term = div_term.float() * (-math.log(10000.0) / d_model)计算 -(2i/d)×ln(10000)，为 exp 做准备
div_term = torch.exp(div_term)exp(-2i/d × ln(10000)) = 10000^(-2i/d) = 分母
pe[:, 0::2] = torch.sin(position * div_term)所有偶数位置（0,2,4,…）填 sin(pos / 分母)
pe[:, 1::2] = torch.cos(position * div_term)所有奇数位置（1,3,5,…）填 cos(pos / 分母)
return pe返回位置编码矩阵，shape=(seq_len, d_model)

概念三：可学习位置编码 —— 另一种思路

正弦编码很优雅，但GPT-2 实际上用的不是它——GPT-2 用的是可学习位置编码（Learnable Position Embedding）：

  • 不靠 sin/cos 公式计算，而是直接把位置编码当成可以训练的 Embedding 表
  • 初始随机，在训练中和词嵌入一起被梯度下降优化
  • 本质上就是另一个 nn.Embedding
pos_embed = nn.Embedding(max_seq_len, d_model)位置也是 Embedding！max_seq_len 行 × d_model 列
# 例如 max_seq_len=1024, d_model=768能处理最长 1024 个 token 的序列
pos_ids = torch.arange(seq_len)位置 ID：[0, 1, 2, …, seq_len-1]
pos_emb = pos_embed(pos_ids)查表：位置 0 → 第 0 行，位置 1 → 第 1 行...
# pos_emb.shape → (seq_len, d_model)和 token embedding 形状完全一样
正弦位置编码可学习位置编码
需要训练吗？❌ 不需要（直接算）✅ 需要（随训练更新）
能外推到更长序列？✅ 理论上可以❌ 不行（最多到 max_seq_len）
效果够用，但非最优略好（训练适配数据）
GPT-2 用哪个？—✅ 可学习
实现复杂度需要写 sin/cos 函数一行 nn.Embedding

本课程两种都教你。实际项目中 GPT-2 用可学习编码，但了解正弦编码有助于理解「位置信息如何被编码成向量」。

概念四：组装 —— token_emb + pos_emb

加法，不是拼接

有了词嵌入和位置编码，下一步就是把它们合并起来。 最常见的做法是：

x = token_embedding + position_embedding

等等，为什么是加法？ 如果 tok_emb 的 shape 是 (seq_len, 768)，pos_emb 的 shape 也是 (seq_len, 768)， 加法就是对应位置的数字直接相加。但这不会把「词信息」和「位置信息」搞混吗？

不会——因为加法把两种信息混合在了同一个向量的不同维度上。 在后续的 Transformer 层处理中，模型通过训练学会了： 「这些维度模式来自词义，那些模式来自位置」。

为什么不用拼接（concatenation）？拼接会得到 (seq_len, 1536) 的向量，维度翻倍， 后面所有层的参数量也会翻倍。加法不改变维度，且效果等价——这是反复工程验证后的最优选择。

Dropout：防止「死记硬背」

Dropout 是训练中的一个简单技巧：随机把一些值设为零。 在输入层（Embedding 之后）通常加一个 Dropout(0.1)：

dropout = nn.Dropout(0.1)以 10% 的概率，随机把某些值设为零
x = dropout(tok_emb + pos_emb)每轮训练随机关掉 10% 的值——防止模型依赖特定位置

为什么有效？想象你在背课文——如果每次读的时候随机遮住一些字，你就被迫理解整体意思而不是死记笔画。 Dropout 强迫模型学习更鲁棒的表示，不能依赖任何一个特定的值。训练完后推理时，Dropout 自动关闭。

完整代码：GPT 的输入处理层

下面是把所有概念拼在一起的完整代码——这是 GPT 模型的「第一段」：

import torch
import torch.nn as nn
import math

class GPTInputLayer(nn.Module):
    """GPT 的输入处理层：Token ID → 词嵌入 + 位置编码 → Dropout"""

    def __init__(self, vocab_size=50257, d_model=768, max_seq_len=1024, dropout=0.1):
        super().__init__()

        # --- 词嵌入：一张 50257 × 768 的大表 ---
        self.tok_embed = nn.Embedding(vocab_size, d_model)

        # --- 位置编码：一张 1024 × 768 的表（可学习） ---
        self.pos_embed = nn.Embedding(max_seq_len, d_model)

        # --- Dropout：防止过拟合 ---
        self.dropout = nn.Dropout(dropout)

    def forward(self, token_ids):
        """
        token_ids: shape (batch, seq_len)，例如 (2, 512) 表示 2 句话，每句 512 个 token

        返回：shape (batch, seq_len, d_model) 即 (2, 512, 768)
        """
        batch_size, seq_len = token_ids.shape

        # Step 1: 查词嵌入表
        tok_emb = self.tok_embed(token_ids)           # (batch, seq_len, d_model)

        # Step 2: 查位置编码表
        pos_ids = torch.arange(seq_len)                # [0, 1, 2, ..., seq_len-1]
        pos_emb = self.pos_embed(pos_ids)              # (seq_len, d_model)

        # Step 3: 加法融合 ← 核心！词信息 + 位置信息
        x = tok_emb + pos_emb                          # (batch, seq_len, d_model)

        # Step 4: Dropout（只在训练时生效，推理时自动跳过）
        x = self.dropout(x)

        return x


# ===== 测试：随机输入，检查输出 shape =====
model = GPTInputLayer()

print(f"参数量: {sum(p.numel() for p in model.parameters()):,}")
