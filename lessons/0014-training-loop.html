🔄 第14课 · 训练循环实战

训练 = 反复做五件事

L6 讲了你训练循环的原理——前向→损失→反向→更新。L12 有了模型，L13 有了数据。 现在把一切串起来，写出能真正降低 loss 的训练脚本。

取一个 batch
inputs, targets
→
前向传播
model(inputs) → logits
→
算损失
cross_entropy(logits, targets)
→
反向传播
loss.backward()
→
更新参数
optimizer.step()

循环第①步。训练就是这五步的无限重复——直到 loss 不再下降。

概念一：loss 计算 —— 模型错得有多离谱

L6 讲过的交叉熵损失（Cross-Entropy Loss）——衡量预测概率分布和正确答案之间的差距。 PyTorch 自带： 

loss_fn = nn.CrossEntropyLoss()

# logits: (batch, seq_len, vocab_size)  — 模型输出的原始分数
# targets: (batch, seq_len)              — 正确答案的 token ID
loss = loss_fn(logits.view(-1, vocab_size), targets.view(-1))

# .view(-1, vocab_size) 把 batch 和 seq_len 两维合并
# 因为 CrossEntropyLoss 期望输入是 (N, classes) 和 (N,)
# N = batch × seq_len = 所有位置一起算

概念二：优化器 —— 谁在「推」参数

你可能还记得 L6 的梯度下降：参数 = 参数 - 学习率 × 梯度。 PyTorch 的优化器自动做这件事，而且做得更好（AdamW 是目前最主流的选择）： 

optimizer = torch.optim.AdamW(
    model.parameters(),
    lr=6e-4,              # 学习率
    betas=(0.9, 0.95),    # 动量参数（来自 GPT-3 论文）
    weight_decay=0.1      # 权重衰减（L2 正则化）
)
优化器特点
SGD最基础，每个参数统一学习率
Adam每个参数自适应学习率 + 动量
AdamWAdam + 正确的权重衰减，训练大模型首选

概念三：学习率调度 —— 从大火到小火慢炖

训练开始时用大学习率快速进步，后期用小学习率精细调整——像煮汤一样，先大火烧开，再小火慢炖。 

# Warmup：前 100 步从 0 线性增长到 lr（防止一开始梯度不准把参数拽飞）
# Cosine Decay：之后按余弦曲线平滑下降到目标学习率
scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
    optimizer, T_max=1000, eta_min=1e-6
)

概念四：梯度裁剪 —— 防止「砰」一下参数飞走

有时某一个 batch 恰好包含很难预测的文本，梯度会异常大——如果直接用它更新，参数会被「炸」到一个离谱的位置。 梯度裁剪就是给梯度设一个上限： 

# 如果梯度的「模长」超过 1.0，就把它缩回 1.0
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

这个操作在 loss.backward() 和 optimizer.step() 之间执行。

完整训练脚本

import torch
import torch.nn as nn
from torch.utils.data import DataLoader
import time

def train_one_epoch(model, dataloader, optimizer, loss_fn, device='cpu'):
    """训练一个 epoch（遍历全部数据一次）"""
    model.train()
    total_loss = 0
    total_tokens = 0
    start = time.time()

    for step, (inputs, targets) in enumerate(dataloader):
        inputs = inputs.to(device)        # 搬到 GPU（如果有的话）
        targets = targets.to(device)

        # ---- 核心五步 ----
        optimizer.zero_grad()             # ① 清空上一步的梯度
        logits = model(inputs)            # ② 前向传播
        loss = loss_fn(                    # ③ 算损失
            logits.view(-1, logits.size(-1)),
            targets.view(-1)
        )
        loss.backward()                   # ④ 反向传播（自动求梯度）
        torch.nn.utils.clip_grad_norm_(   # 裁剪梯度
            model.parameters(), 1.0)
        optimizer.step()                  # ⑤ 更新参数

        total_loss += loss.item()
        total_tokens += inputs.numel()

        # 每 100 步打印一次进度
        if step % 100 == 0:
            print(f"  Step {step:5d} | loss {loss.item():.4f}")

    elapsed = time.time() - start
    avg_loss = total_loss / len(dataloader)
    tokens_per_sec = total_tokens / elapsed
    return avg_loss, tokens_per_sec


@torch.no_grad()  # 评估时不需要计算梯度
def evaluate(model, dataloader, loss_fn, device='cpu'):
    """在验证集上评估模型"""
    model.eval()
    total_loss = 0

    for inputs, targets in dataloader:
        inputs, targets = inputs.to(device), targets.to(device)
        logits = model(inputs)
        loss = loss_fn(
            logits.view(-1, logits.size(-1)),
            targets.view(-1)
        )
        total_loss += loss.item()

    return total_loss / len(dataloader)


def train(model, train_loader, val_loader, epochs=10, lr=6e-4, device='cpu'):
    """完整训练流程"""
    model = model.to(device)
    optimizer = torch.optim.AdamW(model.parameters(), lr=lr)
    scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(
        optimizer, T_max=epochs * len(train_loader))
    loss_fn = nn.CrossEntropyLoss()

    best_val_loss = float('inf')

    for epoch in range(epochs):
        print(f"\n{'='*50}")
        print(f"Epoch {epoch+1}/{epochs}")

        # 训练
        train_loss, tps = train_one_epoch(
            model, train_loader, optimizer, loss_fn, device)
        scheduler.step()

        # 验证
        val_loss = evaluate(model, val_loader, loss_fn, device)

        print(f"Train loss: {train_loss:.4f} | Val loss: {val_loss:.4f} | {tps:.0f} tok/s")

        # 保存最好的模型
        if val_loss < best_val_loss:
            best_val_loss = val_loss
            torch.save({
                'epoch': epoch,
                'model_state_dict': model.state_dict(),
                'optimizer_state_dict': optimizer.state_dict(),
                'val_loss': val_loss,
            }, 'best_model.pt')
            print(f"  ✅ 保存最佳模型 (val_loss={val_loss:.4f})")

    print(f"\n训练完成！最佳验证 loss: {best_val_loss:.4f}")

概念五：Checkpoint —— 学会「存档」

训练一个模型可能需要几小时甚至几天。如果中途断电或者想从上次中断的地方继续，就需要checkpoint。 

# --- 保存 ---
torch.save({
    'epoch': epoch,                          # 当前轮次
    'model_state_dict': model.state_dict(),  # 模型参数
    'optimizer_state_dict': optimizer.state_dict(),  # 优化器状态
    'val_loss': val_loss,                    # 当前验证损失
}, 'checkpoint.pt')

# --- 恢复 ---
checkpoint = torch.load('checkpoint.pt')
model.load_state_dict(checkpoint['model_state_dict'])
optimizer.load_state_dict(checkpoint['optimizer_state_dict'])
start_epoch = checkpoint['epoch'] + 1       # 从下一轮继续
⚠️ 为什么连 optimizer 也要保存？
Adam/AdamW 优化器内部有每个参数的「动量」状态（可以理解为每个参数的学习速度历史记录）。 如果不保存这个，恢复训练时优化器从头开始重新积累动量——相当于换了个新教练，浪费了之前的「经验」。
