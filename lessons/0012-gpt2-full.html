最后一步：把所有零件拼起来

从 L8 到 L11，你分别实现了张量运算、输入层、注意力机制、Transformer Block。 这节课把它们组装成一个能接受 token ID 输入、输出概率分布的完整 GPT 模型。

GPT-2 架构总览

输入 Token IDs (B, L)
  ↓
Token Embedding + Position Embedding ← L9
  ↓  (B, L, 768)
Dropout
  ↓
Transformer Block 0 ← L11
Transformer Block 1
  ···
Transformer Block 11 ×12 层
  ↓  (B, L, 768)
LayerNorm (final)
  ↓
Linear → vocab_size 768 → 50257
  ↓
输出 Logits (B, L, vocab_size)

就这些。GPT-2 small（124M 参数版本）没有更多的东西了。 12 层 Block、768 维向量、12 个注意力头——这些数字就是 GPT-2 small 的全部超参数。

概念一：输出层 —— 从向量回到词

Transformer 输出的是 768 维的向量，但我们需要的是「下一个 token 是什么」——需要回到词表空间（50257 维）。 输出层就是最后一个 nn.Linear：

self.out_head = nn.Linear(768, 50257, bias=False)把每个 token 的 768 维向量投影到 50257 维
logits = self.out_head(x)(B, L, 768) → (B, L, 50257)
# logits[0, 5, :] = 第 0 句话第 5 个位置上，50257 个 token 各自的「分数」分数最高的那个 token = 模型预测的下一个 token
💡 一个有趣的设计选择
GPT-2 的输出层权重与输入 Embedding 权重是共享的（Weight Tying）。 这意味着 out_head.weight = tok_embed.weight ——同一个矩阵既用来把 token ID 转成向量（查表）， 也用来把向量转回 token 概率（投影）。这种做法节省了 38M 参数（近 1/3），且效果几乎不下降。

概念二：GPTModel 完整代码

import torch
import torch.nn as nn
import math

class GPTModel(nn.Module):
    """GPT-2 small (124M parameters) 的完整实现"""

    def __init__(self, vocab_size=50257, d_model=768, num_heads=12,
                 num_layers=12, max_seq_len=1024, dropout=0.1):
        super().__init__()

        # --- 1. 输入层（L9） ---
        self.tok_embed = nn.Embedding(vocab_size, d_model)
        self.pos_embed = nn.Embedding(max_seq_len, d_model)
        self.dropout_emb = nn.Dropout(dropout)

        # --- 2. Transformer Blocks（L11）× 12 ---
        self.blocks = nn.ModuleList([
            TransformerBlock(d_model, num_heads, dropout)
            for _ in range(num_layers)
        ])

        # --- 3. 最终 LayerNorm ---
        self.ln_final = nn.LayerNorm(d_model)

        # --- 4. 输出层：向量 → 词表分数 ---
        self.out_head = nn.Linear(d_model, vocab_size, bias=False)

        # --- 参数初始化 ---
        self.apply(self._init_weights)

    def _init_weights(self, module):
        if isinstance(module, nn.Linear):
            torch.nn.init.normal_(module.weight, mean=0.0, std=0.02)
            if module.bias is not None:
                torch.nn.init.zeros_(module.bias)
        elif isinstance(module, nn.Embedding):
            torch.nn.init.normal_(module.weight, mean=0.0, std=0.02)

    def forward(self, token_ids):
        B, L = token_ids.shape

        # Step 1: 词嵌入 + 位置编码
        tok_emb = self.tok_embed(token_ids)                       # (B, L, D)
        pos_ids = torch.arange(L, device=token_ids.device)
        pos_emb = self.pos_embed(pos_ids)                         # (L, D)
        x = self.dropout_emb(tok_emb + pos_emb)                   # (B, L, D)

        # Step 2: 过 12 层 Transformer Block
        for block in self.blocks:
            x = block(x)                                          # shape 不变！

        # Step 3: 最终 LayerNorm
        x = self.ln_final(x)

        # Step 4: 输出投影 → 词表分数
        logits = self.out_head(x)                                 # (B, L, vocab_size)

        return logits


# ===== 测试：随机输入 → 验证 shape =====
model = GPTModel()
dummy_ids = torch.randint(0, 50257, (2, 512))  # 2 句话，每句 512 token
logits = model(dummy_ids)

print(f"输入: {dummy_ids.shape}")   # (2, 512)
print(f"输出: {logits.shape}")      # (2, 512, 50257)
print(f"最后位置预测下一个 token：取 logits[:, -1, :].argmax(dim=-1)")

# 预测下一个 token
next_token = logits[:, -1, :].argmax(dim=-1)
print(f"预测的下一个 token ID: {next_token}")  # 随机（因为还没训练）

# ===== 参数统计 =====
total = sum(p.numel() for p in model.parameters())
print(f"总参数量: {total:,}")        # 约 124M
print(f"可训练参数: {sum(p.numel() for p in model.parameters() if p.requires_grad):,}")
🔗 与 HuggingFace GPT-2 的对应关系
上面的 GPTModel 结构与 HuggingFace 的 GPT2Model 基本一致：
• self.tok_embed ≈ wte（Word Token Embedding）
• self.pos_embed ≈ wpe（Word Position Embedding）
• self.blocks ≈ h（12 层 Block）
• self.ln_final ≈ ln_f
• self.out_head 在 HF 里共享 wte（Weight Tying）

如果你的代码输出和 HuggingFace 的 GPT-2 在相同输入下完全一致——那说明你的实现是正确的。
