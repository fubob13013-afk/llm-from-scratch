🧭 第3课 · 词嵌入与位置编码 —— 给词语赋予「坐标」

第 2 课把文本变成了 Token ID（整数序列）。但整数只是编号，不携带语义。本课解决两个问题：怎么让数字代表含义？怎么让模型知道词序？

⚠️ 本课前提 需要知道什么是 Token ID（第 2 课）。回顾：经过分词和词表映射后，「我喜欢人工智能」变成了类似 [15, 42, 38, 89, 91] 这样的整数序列。

第一部分：词嵌入（Embedding）—— 从整数到语义坐标

问题：Token ID 有什么致命缺陷？

假设词表里：

"国王"  → Token ID = 15
"王后"  → Token ID = 382
"苹果"  → Token ID = 16

从数字上看，「国王」（15）和「苹果」（16）只差 1——比「国王」和「王后」（差 367）「更接近」。但语义上显然「国王」和「王后」更相关。

Token ID 只是一个编号，不携带任何含义。计算机需要的是：把每个词表示成一个能体现语义的「位置」——含义相近的词，位置也相近。

什么是向量？—— 30 秒速通

你可能听过「向量」这个词但觉得抽象。其实它就是一组数字，用来描述一个点在空间中的位置。

🔑 颜色 RGB 类比（你肯定懂这个）
你在 PS 里调过色，知道每个颜色都可以用 (R, G, B) 三个数字表示：
红色
(230, 57, 70)
橙红色
(230, 115, 70)
蓝色
(69, 123, 157)

注意：红色和橙红色的 RGB 值很接近（只有 G 值差了一点），而它们和蓝色的值差得很远。这恰好就是 Embedding 要做的事——相似的东西，坐标也相似。

一个 RGB 颜色是一个 3 维向量。词嵌入做的事完全一样，只是维度更多：不是 3 维，而是 768 维、2048 维甚至 4096 维。

为什么需要这么多维？因为颜色的差异只需要 3 个维度就能描述，但人类语言的语义差异极其复杂——性别、时态、感情色彩、正式程度、领域……需要几百上千个维度来刻画。

2 为什么这能行？—— 分布式语义假说

语言学里有一个著名的观察（不是假说，是大量实验验证过的规律）：

「一个词的含义，由它身边的词决定。」
— John Rupert Firth, 1957

什么意思？

我喝了一杯___。    ← 空格里大概率是「水」「茶」「咖啡」「牛奶」……
我吃了一个___。    ← 空格里大概率是「苹果」「面包」「鸡蛋」……

「水」「茶」「咖啡」经常出现在相同的上下文里（「喝」的后面），所以它们的嵌入向量会很接近。「苹果」「水」虽然都可以出现在空格里，但一个搭配「吃」一个搭配「喝」——它们的向量就会远一些。

模型在训练时，并不是被显式告知「国王和王后有关」。它只是在几万亿次「看到前面猜后面」的训练中，发现了：出现在相似上下文里的词，调整它们的向量让它们靠近，能降低预测误差。

第二部分：位置编码（Positional Encoding）—— 让模型知道词序

Embedding 有一个致命缺陷

Embedding 只给每个词一个固定坐标——「国王」不管出现在句子第 1 位还是第 5 位，查出来的向量都是完全一样的。

但「狗追猫」和「猫追狗」——词完全相同，顺序不同，意思天差地别。只用词嵌入（不加位置信息），模型看到的两个句子是一模一样的输入，无法区分谁追谁。

解决方案：在词嵌入的基础上，再叠加一个只和位置有关的向量——位置编码（Positional Encoding）。

最终输入向量 = 词嵌入向量（"苹果"的语义）+ 位置编码向量（"第 4 个位置"的指纹）

这样同一个词在不同位置会有不同的最终表示。

GPT 用的方法：正弦位置编码

对于位置 pos 和维度 i（0 ≤ i < 768）：

  • 偶数维度（0, 2, 4, …）：sin(pos / 10000i/768)
  • 奇数维度（1, 3, 5, …）：cos(pos / 10000(i-1)/768)

这个公式的直觉：

  • i 越小（靠前的维度）→ 分母接近 1 → sin/cos 的波长很短 → 相邻位置的编码变化很快 → 捕捉局部位置差异
  • i 越大（靠后的维度）→ 分母很大 → sin/cos 的波长很长 → 编码变化缓慢 → 捕捉远距离位置关系

不同频率的 sin/cos 波叠加在一起，每个位置拥有一个独一无二的向量指纹。相邻位置的指纹很相似（帮助模型学到局部模式），远距离位置的指纹差异很大。

可学习 vs 正弦位置编码

方案做法优点缺点
可学习位置编码 给每个位置随机初始化一个向量，训练时调整 灵活，GPT-2 及之后多用此方案 只能处理训练时见过的位置长度
正弦位置编码 用 sin/cos 公式计算，不参与训练 可泛化到任意长度；相邻位置天然相似 固定规则，不如可学习的灵活

原始 Transformer 论文和 GPT-1 用正弦编码。GPT-2 及之后大多改用可学习位置编码。但核心思想完全一样：给每个位置一个独一无二的向量，加到词嵌入上。

🧪 小测验

问题 1/3 — 查表

GPT-2 的 Embedding 表格有多少行？多少列？（给出数量级即可）

点击看答案
约 50,000 行 × 768 列。
行数 = 词表大小（GPT-2 词表约 50,257 个 token），
列数 = 嵌入维度（GPT-2 用 768）。
总数约 3800 万个数字——这些都是模型要学的参数。

问题 2/3 — 位置的作用

「狗追猫」和「猫追狗」——分词后 token 完全相同，只是顺序不同。只用词嵌入（不加位置编码）的话，模型能看到区别吗？为什么？

点击看答案
不能。只用词嵌入的话，「狗」无论在第 1 位还是第 3 位，查出来的向量一模一样。两个句子送入 Transformer 的矩阵完全相同，模型无法区分谁追谁。

这就是为什么必须加位置编码——同一个词在不同位置，最终输入向量不同。

问题 3/3 — 语义坐标的来历

训练好的 Embedding 中，「猫」和「狗」的向量距离近，还是「猫」和「汽车」的向量距离近？这些坐标是人为设定的吗？

点击看答案
「猫」和「狗」更近。这些坐标不是人为设定的，是模型自己学出来的。
因为在训练数据中，「猫」和「狗」频繁出现在相似的上下文里（"我养了一只___"），模型在训练中把它们的向量调到了彼此附近。「汽车」出现在完全不同的上下文里，向量自然离得远。
