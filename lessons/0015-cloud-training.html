☁️ 第15课 · 在云端训练

你没有 GPU？没关系。

训练 GPT 需要 GPU。但你不必买一块——Google Colab 提供免费的 T4 GPU（约 15GB 显存）， 足够训练一个小型 GPT 模型。国内用户也可以用 AutoDL（按小时付费，几毛钱一小时）。

方案 A：Google Colab（免费，需科学上网）

第一步：打开 Colab

  1. 打开 colab.research.google.com
  2. 点击「新建笔记本」
  3. 菜单：修改 → 笔记本设置 → 硬件加速器 → T4 GPU

第二步：检查 GPU

!nvidia-smi

# 应该看到 Tesla T4，15GB VRAM

第三步：安装依赖 + 上传代码

!pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
!pip install tiktoken

# 上传你的 gpt2.py（L12 的完整模型代码）
from google.colab import files
files.upload()  # 选择你的 .py 文件

第四步：跑训练

# 将 Colab 的 GPU 设为 device
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
print(f"使用设备: {device}")

# 加载模型 + 数据 + 训练
model = GPTModel().to(device)
dataloader = create_dataloader("data.txt", batch_size=4)

train(model, dataloader, val_loader=None, epochs=5, device=device)
⚠️ Colab 限制 免费版 GPU 有使用时长限制（通常连续使用数小时后会被断开），且闲置约 30 分钟会自动断开。 务必保存 checkpoint——用 torch.save() 存到 Google Drive 或下载到本地。

方案 B：AutoDL（国内，按量付费）

如果没法科学上网，AutoDL 是国内最好的选择：

  1. 打开 autodl.com，注册并充值 10 元（够用很久）
  2. 选一个实例：推荐 RTX 3080（10GB 显存），约 1 元/小时
  3. 选镜像：PyTorch 2.x + Python 3.10
  4. 启动后通过 JupyterLab 或 SSH 连接，上传代码和数据
  5. 和 Colab 一样运行训练脚本
💡 AutoDL 小技巧 • 使用「无卡模式」开机（0.1 元/小时）先上传数据、调试代码
• 一切就绪后再切到 GPU 实例跑训练
• 训练完立即关机，按实际使用时间计费
• 数据存在 /root/autodl-tmp/（系统盘，不另收费）

训练监控：看什么指标？

指标正常表现异常表现及原因
训练 loss持续下降，前期快后期慢不降→学习率太小或代码bug；震荡→学习率太大
验证 loss跟随训练 loss 下降不降反升→过拟合；远高于训练loss→数据量不足
训练速度几千~几万 token/秒太慢→batch 太小或未用 GPU
GPU 利用率80%~100%很低→batch 太小或数据加载成了瓶颈

训练多久？预期什么结果？

以小模型（d_model=768, 12层, 124M参数）+ 小数据（~50MB）为例：

  • 1 epoch（约 1-2 小时）：loss 从 ~11 降到 ~6——模型还在学基础知识
  • 3 epochs（约 4-6 小时）：loss 降到 ~4——模型开始生成通顺但不一定准确的文本
  • 10 epochs（约 15-20 小时）：loss 降到 ~2——模型过拟合训练数据，但对验证集仍有相当效果
🎯 关键心态 你的第一个模型大概率不会生成惊艳的文本——这完全正常。GPT-2 small 本身就不是为中文优化的， 加上数据量有限，产出会是一个「能说人话但经常胡说八道」的模型。

成功的标准不是模型有多好——而是 loss 在下降。只要 loss 在降，就说明你的代码是正确的。 这就是 Phase 3 的目标：验证你从零写的 GPT 能学会语言模式。
