概念一：Q/K/V 投影 —— 一个输入，三个分身

注意力机制的第一个操作：把输入向量 x（768 维）分别投影到查询（Q）、键（K）、值（V）三个空间。 投影就是矩阵乘法——也就是 nn.Linear，你已经在 L8 见过了。

self.W_query = nn.Linear(d_model, d_model, bias=False)Q 投影矩阵：768 → 768，bias=False 是 GPT-2 的设计选择
self.W_key = nn.Linear(d_model, d_model, bias=False)K 投影矩阵：每个 Linear 有自己独立的权重
self.W_value = nn.Linear(d_model, d_model, bias=False)V 投影矩阵：三个矩阵形状相同，但数值不同（各自学习）
Q = self.W_query(x) # (B, L, d) → (B, L, d)每个 token 的向量 × W_Q → 得到该 token 的「查询」
K = self.W_key(x) # 同理，得到「键」Q 和 K 的投影矩阵不同（W_Q ≠ W_K），打破对称性
V = self.W_value(x) # 得到「值」V 是被加权求和的对象，携带最终输出的语义
🔗 回到 L4 Q 是「我要找什么」，K 是「我有什么标签」，V 是「我有什么内容」。 三个 Linear 各自学习——训练后，W_Q 学会生成「会提问」的向量，W_K 学会生成「容易被找到」的向量。

概念二：Q @ Kᵀ —— 计算「谁该关注谁」

有了 Q 和 K，接下来计算注意力分数矩阵——每个 token 对每个 token 的「关注程度」。 核心操作只有一行： 

attn_scores = Q @ K.transpose(-2, -1) / math.sqrt(d_k)

拆解这一行：

部分含义Shape
Q查询矩阵(B, num_heads, L, d_k)
K.transpose(-2, -1)把 K 的最后两维转置(B, num_heads, d_k, L)
Q @ K.T矩阵乘法 → 每对 token 的点积分数(B, num_heads, L, L)
/ sqrt(d_k)除以 √d_k（缩放因子）防止点积太大导致 Softmax 饱和

结果是一张 L×L 的方阵，其中第 i 行第 j 列的值 = 第 i 个 token 的 Q 与第 j 个 token 的 K 的点乘结果（缩放后）。 值越大 = 第 i 个 token 越应该关注第 j 个 token。

💡 为什么除以 √d_k？
点积的方差会随维度 d_k 增大而增大（因为累加了大量乘积）。 Q·K 里的值可能变得很大（比如 ±50），Softmax 会对大值输出接近 0 或 1 的极端分布—— 这会导致梯度消失，训练不动。除以 √d_k 把方差压回 1，保持 Softmax 在「温和」区间。
对于 GPT-2：d_k = 64，√d_k = 8。

概念三：因果遮罩 —— 不准偷看未来

GPT 是自回归模型：预测第 5 个词时，只能看第 1~4 个词，不能看第 5 个及之后。 如果不加限制，模型看到答案再答题，训练等于作弊。

实现方法：生成一个下三角矩阵（torch.tril），把上三角（未来位置）设为 -inf。

# 生成 L×L 的下三角矩阵（下三角=1，上三角=0）
mask = torch.tril(torch.ones(L, L))

# 把上三角的注意力分数设为 -inf
# Softmax(-inf) = 0 —— 意味着完全不能关注
attn_scores = attn_scores.masked_fill(mask == 0, float('-inf'))

attn_weights = torch.softmax(attn_scores, dim=-1)   # (B, heads, L, L)

概念四：多头注意力 —— 12 个视角，同时看

单头注意力只能学到一种「关注模式」——比如总是关注相邻的 token。 但语言需要多种关注模式：语法头关注主谓搭配，语义头关注同义词， 距离头关注远处的修饰关系……

多头 = 把 d_model 切成 num_heads 份，每份独立做注意力，最后拼回来。

GPT-2 small 的参数：d_model = 768，num_heads = 12，d_k = 768 / 12 = 64。

拆头和拼回的操作

# 输入: x shape = (B, L, 768)以 GPT-2 small 为例
B, L, D = x.shape
# Step 1: Q 投影后 reshape → 拆成 12 个头
Q = self.W_query(x) # (B, L, 768)先投影
Q = Q.view(B, L, 12, 64) # 把 768 切成 12×64最后一维被切成了 12 段，每段 64
Q = Q.transpose(1, 2) # (B, 12, L, 64)把 head 维度提到前面，方便并行
# Step 2: 每个头独立做注意力（K, V 同理）
attn = Q @ K.T / sqrt(64) # (B, 12, L, L)12 个头各自算 QKᵀ，互不干扰
attn = masked_softmax(attn) 每个头各自遮罩 + 归一化
out = attn @ V # (B, 12, L, 64)每个头各自加权 V
# Step 3: 把 12 个头拼回来
out = out.transpose(1, 2) # (B, L, 12, 64)把 head 维度放回中间
out = out.reshape(B, L, 768)# (B, L, 768)12×64 拼回 768
out = self.out_proj(out) # 最后再过一个 Linear 融合多头信息W_O 投影：把多头拼起来的结果再加工一次
🔗 回到 L5 L5 里说过「多头 = 12 个单头并行」。上面的 view + transpose 就是把原理变成代码的具体操作。 注意：每个头只处理 64 维（不是 768），这样 12 个头加起来和单头 768 的计算量基本相同。

完整代码：MultiHeadAttention 类

import torch
import torch.nn as nn
import math

class MultiHeadAttention(nn.Module):
    """GPT-2 的多头自注意力模块"""

    def __init__(self, d_model=768, num_heads=12, dropout=0.1):
        super().__init__()
        assert d_model % num_heads == 0, "d_model 必须被 num_heads 整除"
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads          # 每个头的维度：768/12 = 64

        # --- Q/K/V 投影矩阵（合并成一个大的，再切分） ---
        self.W_query = nn.Linear(d_model, d_model, bias=False)
        self.W_key   = nn.Linear(d_model, d_model, bias=False)
        self.W_value = nn.Linear(d_model, d_model, bias=False)

        # --- 输出投影：把多头拼起来后再加工 ---
        self.out_proj = nn.Linear(d_model, d_model, bias=False)

        # --- Dropout：对注意力权重做正则 ---
        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        B, L, D = x.shape   # batch, seq_len, d_model

        # 1. 投影到 Q/K/V 空间
        Q = self.W_query(x)  # (B, L, D)
        K = self.W_key(x)
        V = self.W_value(x)

        # 2. 拆成多头：(B, L, D) → (B, num_heads, L, d_k)
        Q = Q.view(B, L, self.num_heads, self.d_k).transpose(1, 2)
        K = K.view(B, L, self.num_heads, self.d_k).transpose(1, 2)
        V = V.view(B, L, self.num_heads, self.d_k).transpose(1, 2)

        # 3. 点积注意力：Q @ K.T / sqrt(d_k)
        attn_scores = Q @ K.transpose(-2, -1) / math.sqrt(self.d_k)
        # shape: (B, num_heads, L, L)

        # 4. 因果遮罩
        mask = torch.tril(torch.ones(L, L)).to(x.device)
        attn_scores = attn_scores.masked_fill(mask == 0, float('-inf'))

        # 5. Softmax + Dropout
        attn_weights = torch.softmax(attn_scores, dim=-1)
        attn_weights = self.dropout(attn_weights)

        # 6. 加权求和 V
        out = attn_weights @ V   # (B, num_heads, L, d_k)

        # 7. 拼回多头 → 输出投影
        out = out.transpose(1, 2).reshape(B, L, D)   # (B, L, D)
        out = self.out_proj(out)

        return out


# ===== 测试 =====
mha = MultiHeadAttention(d_model=768, num_heads=12)
x = torch.randn(2, 512, 768)      # 模拟：2 句话，每句 512 token，每个 768 维
out = mha(x)
print(f"输入 shape: {x.shape}")     # (2, 512, 768)
print(f"输出 shape: {out.shape}")   # (2, 512, 768) — 和输入一模一样！
📊 关键洞察：输入输出 shape 相同 注意力模块的输入和输出 shape 完全一致：(B, L, 768) → (B, L, 768)。 这不是巧合，而是刻意设计——因为 Attention 和 FFN 要穿插使用，shape 不变才能叠加残差连接。 你将在 L11 看到这一点。
