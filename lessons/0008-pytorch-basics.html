PyTorch 是什么？

在 Phase 1，你学的是「原理」——用大白话和图示理解 Transformer 怎么工作。 从这节课开始，你要学的是「实现」——用代码把那些原理变成真正能跑的模型。

写代码需要工具。就像做设计需要 Photoshop、写论文需要 Word，写深度学习模型需要 PyTorch。

📦 PyTorch 是什么？
PyTorch 是一个深度学习框架（Deep Learning Framework）——一套专门为神经网络设计的 Python 库。 它由 Meta（原 Facebook）的 AI 研究团队在 2016 年发布，现在是全世界最流行的深度学习框架之一。

你可以把它理解成一个工具箱：里面有处理数据的容器（张量）、有搭建网络的积木（nn 模块）、有自动算梯度的引擎（autograd）。 没有 PyTorch，你也能写神经网络——但那相当于用石头砸钉子，又慢又容易出错。PyTorch 就是那把锤子。

为什么选 PyTorch？ 三个原因：

  • Python 原生体验——写 PyTorch 就像写普通的 Python 代码，不像某些框架需要学一门新语言
  • 研究社区首选——绝大多数 AI 论文（包括 GPT 系列）都用 PyTorch 实现，学会它你就能看懂最新的开源代码
  • GPU 加速透明——同样的代码，移到 GPU 上跑通常只改一行，PyTorch 自动处理底层细节

PyTorch 提供了哪三样东西？

Phase 1 的 7 节课，你已经看懂了 Transformer 的每一个零件：Embedding 查表、 Q/K/V 注意力、残差连接、梯度下降……但「看懂」和「写出来」之间还有一道坎—— 你需要一个工具，把数学概念翻译成能跑的代码。

PyTorch 就是这个翻译官。 它提供了三样东西，对应我们接下来的所有工作：

概念一：张量（Tensor）= 多维数组

快递盒直觉 —— 张量到底是什么？

在 Python 里你用过数字、列表。张量就是这些概念的统一升级版——它给数据加上了「形状（shape）」的概念，而且能一次性处理整个数组。

张量 = 一个装了数字的盒子，盒子的「形状（shape）」告诉你每个方向上有几个数字。
📦
张量 = 盒子
一个容器，里面装着很多数字
📐
Shape = 尺寸
盒子的长宽高，告诉你怎么排列数字
⚡
GPU = 叉车
知道尺寸就能一次搬走整个盒子，而不是一个一个捡

这就是为什么「给数据加上形状」就能一次性处理整个数组：GPU 知道这堆数字的排列方式后，可以同时对所有数字做运算——而不是像 CPU 那样一个接一个。速度差 100 倍以上。

从标量到 N 维 —— 一步步升级

有了「盒子」的直觉，我们来看看具体有哪些尺寸的盒子。下面从最简单的标量开始，一级一级往上走。每升一级，就是外面多套一层方括号。

0 维 —— 标量（Scalar）

就是一个单独的数字。没有行列，没有方向，就一个数。

42

Python: 42 ｜ Shape: ()

等一下——你可能会在别处看到标量的 shape 写成 (1,)。它和 () 是同一个东西吗？

不是。()（空括号）是真正的 0 维标量——torch.tensor(42).shape 返回 torch.Size([])。而 (1,) 是一个1 维张量，只不过长度刚好是 1——torch.tensor([42]).shape 返回 torch.Size([1])。区别在于你有没有用方括号把它包起来：裸数字 42 是标量（0 维），[42] 是含一个元素的列表（1 维）。就像一个孤立的苹果，和一个只装了一个苹果的袋子——前者没有「容器」的概念。

1 维 —— 向量（Vector）

把几个标量排成一条线，就有了向量。shape (3,) 表示「这一条线上有 3 个数」。

Python: [1, 2, 3] ｜ Shape: (3,)

你发现了吗？Python 写的是 [1, 2, 3]（方括号），shape 写的是 (3,)（圆括号）。它们是什么关系？

[1, 2, 3] 是值本身——一个 Python list，里面装着 1、2、3 这三个数。而 (3,) 是对结构的描述——告诉你「第一个维度的大小是 3」。打个比方：鞋盒标签上写着「尺码 42」——[鞋, 鞋垫, 鞋带] 是盒子里装的三样东西（值），(3,) 只是告诉你「这一维的尺寸是 3」。shape 是标签，list 是内容物，两回事。

⚠ 注意：「三维向量」里的「维」指的是这条线上有 3 个数，不是「3 维空间」的意思。向量的「维」= 数的个数，张量的「维」= shape 里有几个数字。两个「维」是同一个意思：都是数一数有几个独立的方向。

4 维、5 维……N 维

3 维以上人脑就画不出来了，但规则完全一样——每多一维，就是在最外面多套一层方括号。

LLM 中最常见的形状 —— (batch, seq_len, d_model)

速查表最后一行不是随便写的。(batch, seq_len, d_model) 是整个 Transformer 架构中最核心的数据形状——从 Embedding 查表到注意力计算到 FFN 输出，数据始终保持着这个三维结构在模型里流动。下面把它拆开来看：

字母全称含义举例
batch 批次大小 同时处理几句话。GPU 不吃独食——一次喂它一批句子，它并行处理，速度快几十倍。 32
seq_len 序列长度 每句话最多几个 token。句子有长有短，短的用 padding（填充符）补到一样长，这样才能叠成一个整齐的矩阵。 512
d_model 向量维度 每个 token 用多少个数字来表示。一个 token 不是一个数字，而是一个很长的向量（比如 768 个数字），用来编码这个词的语义、语法、上下文信息。 768

可视化：shape = (2, 5, 768) 的张量长什么样？

batch=0 —— 句子 "我爱北京"（3 个 token + 2 个填充位 ⬚）
位置 0 → "我" → [0.12, -0.34, 0.56, ... 共 768 个数]
位置 1 → "爱" → [-0.23, 0.78, -0.11, ... 共 768 个数]
位置 2 → "北京" → [0.45, -0.09, 0.33, ... 共 768 个数]
位置 3 → ⬚ → [0, 0, 0, ... 共 768 个 0]（填充位）
位置 4 → ⬚ → [0, 0, 0, ... 共 768 个 0]（填充位）
batch=1 —— 句子 "天安门上太阳升"（5 个 token，不需要填充）
位置 0 → "天安" → [0.67, -0.52, 0.18, ... 共 768 个数]
位置 1 → "门" → [-0.41, 0.33, -0.75, ... 共 768 个数]
位置 2 → "上" → [0.29, 0.61, -0.44, ... 共 768 个数]
位置 3 → "太阳" → [-0.18, -0.27, 0.82, ... 共 768 个数]
位置 4 → "升" → [0.55, 0.13, -0.39, ... 共 768 个数]

总元素数 = 2 × 5 × 768 = 7,680 个数字。GPU 一次性处理这 7,680 个数字，而不是一句一句、一个词一个词地来。

为什么需要 batch？如果没有 batch（即 batch=1），GPU 一次只处理 1 句话，大部分时间浪费在等待下一批数据。有了 batch（比如 32），GPU 一次处理 32 句话，计算单元满负荷运转，训练速度提升 30 倍以上。这就是为什么大模型训练要用 GPU 集群——不是为了炫富，而是batch 越大，训练越快。

调整维度参数，观察张量的结构变化。每个小格子代表张量里的一个数。

shape: (3, 4)

在 PyTorch 里创建张量和创建 Python 列表一样简单，但多了一个「形状」的意识。下面逐行拆解：

x = torch.randn(2, 3, 4) randn = random normal（标准正态分布随机数）。
参数 (2, 3, 4) 就是我们要的 shape：
   第 1 个数字 2 → 最外层有 2 个矩阵
   第 2 个数字 3 → 每个矩阵有 3 行
   第 3 个数字 4 → 每行有 4 个数
就像订制一个盒子：「长 2、宽 3、高 4」（然后里面随机填数字）。
x.shape .shape 是张量的尺寸属性（不是方法，不用加括号）。
返回 torch.Size([2, 3, 4]) —— 这是一个特殊的元组，告诉你「这个盒子长什么样」。
类比：快递盒标签上印的「长×宽×高」。
对比 Python list：len(lst) 只能告诉你最外层有几个元素；
.shape 告诉你每一层的尺寸，不管多少维。
x[0] 索引规则：每用一个 [n]，就剥掉最外面一层。
x 的 shape = (2, 3, 4)：
   最外层有 2 个东西 → x[0] 取第 1 个 → 剥掉最外层
   剩下来的是 一个 shape = (3, 4) 的矩阵（3 行 × 4 列）
图示：(2, 3, 4) → 去掉 2 这一维 → 剩下 (3, 4)
x[0, 1] 连续索引：依次剥掉前两层的皮。
x 的 shape = (2, 3, 4)：
   x[0] 剥掉第一维 → 得到 shape (3, 4) 的矩阵
   [1] 再剥掉第二维 → 取这个矩阵的 第 2 行（索引从 0 开始）
   剩下来的是 一个 shape = (4,) 的向量（4 个数排成一条线）
图示：(2, 3, 4) → 去掉前两维 → 剩下 (4,)
🧠 索引口诀 张量的索引 = 逐层剥皮。
shape (A, B, C, D)：
   x[i] → 剥掉 A 层 → shape 变成 (B, C, D)
   x[i, j] → 剥掉 A 和 B 层 → shape 变成 (C, D)
   x[i, j, k] → 剥掉 A、B、C 层 → shape 变成 (D,)
用了几个索引，就剥掉几层。剩下的 shape 就是结果。

概念二：张量运算 —— 矩阵乘法是核心

张量上的运算有很多，但在 Transformer 里最核心的就三个：

torch.randn(d1, d2, …) — 生成随机张量
生成服从标准正态分布（均值0，方差1）的随机数。初始化模型参数时用它。
torch.matmul(A, B) — 矩阵乘法
这是整个 Transformer 里出现次数最多的运算。Q×KT、注意力分数×V、 WQ×输入……全都是矩阵乘法。理解了它，就理解了计算的核心。
torch.softmax(x, dim=-1) — Softmax 归一化
把任意一串数变成「概率分布」（和为 1）。L4 注意力里的核心操作，一模一样。

插播：什么是「点乘」？ 🎯

在矩阵乘法和注意力的讲解中，你会反复听到「点乘」这个词。它是整个大模型数学里出现频率最高的运算。 不搞懂它，后面的东西就都像在看天书。

一句话定义

点乘 = 对应位置相乘，再把所有乘积加起来。
两个等长向量 → 一个数字（标量）。

手算一个例子

假设有两个 3 维向量：

v1 = [2, 3, 1]      （可以想象成「查询向量 Q」）
v2 = [4, 0, 2]      （可以想象成「被查向量 K」）

第一步：对应位置相乘

位置 0:  2 × 4 = 8
位置 1:  3 × 0 = 0
位置 2:  1 × 2 = 2

第二步：把乘积全部加起来

点乘结果 = 8 + 0 + 2 = 10

所以 v1·v2 = 10。一个数字，来自两个向量的「协同工作」。

⚠️ 常见误区：别把向量的「值」当成「形状」
看到 [2, 3, 1] 时，你可能会想：「这是不是指 2 列 × 3 行 × 1 层的一个立体？」
不是的。这里 [2, 3, 1] 就是三个具体的数值——就像你三个朋友的身高 [172, 165, 180]。
「三维向量」的「维」指「列表里有几个数」，不是「几维空间」或「几行几列」。
向量括号里写的是值本身，不是形状。形状是张量（矩阵）才有的概念。

这个数字是什么意思？

点乘结果的大小反映了两个向量的「方向一致性」：

  • 结果大（正数） → 两个向量指向相似方向 → 它们「相关」
  • 结果接近 0 → 两个向量几乎垂直 → 它们「无关」
  • 结果负 → 两个向量指向相反方向 → 它们「负相关」

在注意力机制（L4）里，Q·K 的点乘结果就是「相关性分数」—— 「猫」的 Q 和「喜欢」的 K 点乘结果大，说明「猫」这个词需要关注「喜欢」。

从一次点乘到批量点乘

点乘是两个向量之间的事。但实际场景中——比如注意力机制——我们需要同时算很多对向量之间的点乘。

举个例子：假设你有 2 个「查询」向量和 2 个「被查」向量：

查询 1: q₁ = [1, 2, 3]      被查 1: k₁ = [7,  9, 11]
查询 2: q₂ = [4, 5, 6]      被查 2: k₂ = [8, 10, 12]

你想知道每个查询跟每个被查之间的相关性——那就需要 2×2 = 4 次点乘： 

q₁·k₁ = 1×7  + 2×9  + 3×11 =  58    ← 查询 1 对被查 1
q₁·k₂ = 1×8  + 2×10 + 3×12 =  64    ← 查询 1 对被查 2
q₂·k₁ = 4×7  + 5×9  + 6×11 = 139    ← 查询 2 对被查 1
q₂·k₂ = 4×8  + 5×10 + 6×12 = 154    ← 查询 2 对被查 2

四次手算太烦了。但如果把查询叠成一个矩阵（每行一个查询）、被查向量也叠成一个矩阵（每列一个被查）—— 一次矩阵乘法就能把四个结果全算出来： 

      A (2×3, 每行=一个查询)         B (3×2, 每列=一个被查)         C (2×2, 所有配对结果)
      ┌           ┐              ┌         ┐              ┌             ┐
      │ 1   2   3 │              │ 7   8   │              │ 58   64  │
      │ 4   5   6 │      ×       │ 9  10   │      =       │ 139  154 │
      └           ┘              │ 11 12   │              └             ┘
      第 1 行 = q₁                └         ┘              每个格子 = 一次点乘
      第 2 行 = q₂               第 1 列 = k₁
                                 第 2 列 = k₂

为什么是「行 × 列」？

刚才的例子已经给出了答案：

  • A 的每一行就是一个独立的向量（比如一个「查询」）
  • B 的每一列就是一个独立的向量（比如一个「被查的键」）
  • 行 · 列 = 一个查询向量 · 一个键向量 = 一次点乘

所以 C[i][j] 就是「第 i 个查询 和 第 j 个键 的相关性分数」。
不是行×行、也不是列×列——因为行是查询、列是键，你想要的恰好是「每个查询 vs 每个键」。

为什么 A 的列数必须等于 B 的行数？

因为点乘要求两个向量长度相同。

A 的一行是查询向量，它有 n 个数 → 点乘的另一方也必须有 n 个数。
B 的一列是键向量，它有 m 个数 → 点乘的另一方也必须有 m 个数。
要让它俩能做点乘，必须有 n = m。也就是：

A 的列数（每行有几个数）= B 的行数（每列有几个数）

在上面例子中：A 是 2×3，B 是 3×2，共享的 3 就是向量长度。如果 A 是 2×4 而 B 是 3×2，那 A 的行有 4 个数、B 的列只有 3 个数——对不上，没法点乘。

4 次点乘，分别对应哪 4 对？

用具体数字再看一遍结果矩阵 C 里的 4 个格子：

C[0][0] = A 第 1 行 · B 第 1 列  =  [1,2,3]·[7,9,11]   = 1×7+2×9+3×11  = 58   (查询1 vs 被查1)
C[0][1] = A 第 1 行 · B 第 2 列  =  [1,2,3]·[8,10,12]  = 1×8+2×10+3×12 = 64   (查询1 vs 被查2)
C[1][0] = A 第 2 行 · B 第 1 列  =  [4,5,6]·[7,9,11]   = 4×7+5×9+6×11  = 139  (查询2 vs 被查1)
C[1][1] = A 第 2 行 · B 第 2 列  =  [4,5,6]·[8,10,12]  = 4×8+5×10+6×12 = 154  (查询2 vs 被查2)

每一个 C[i][j] 就是一次点乘。矩阵乘法 = 把一堆点乘组织成表格，一次算完。

整个 Attention 核心就这三步。PyTorch 里两行代码搞定。

概念三：nn.Linear —— 神经网络最基本的运算单元

先搞清楚：什么叫"线性层"？

"线性"（Linear）在数学里的意思很朴素——只有乘法和加法，没有别的花活。 一个线性层做的事情，用一句话就能说清楚：

输入一串数字 → 乘上一个矩阵 → 加上一个偏移 → 输出另一串数字

写成公式就是整个神经网络里出现次数最多的式子：

y = Wx + b

四个字母，逐个拆开：

符号叫什么是什么打个比方
x 输入（Input） 一串数字。比如一个词的向量有 768 个数，x 就是这个向量 原材料——进工厂之前的钢材
W 权重（Weight） 一个矩阵，里面每个数字都是模型要学的参数。训练过程中不断调整的就是它 工厂里的机器——决定怎么加工
b 偏置（Bias） 一串数字，给每个输出位置加一个"基础偏移"。没有它，当 x 全是 0 时输出也只能是 0 机器的默认设置——没原料时也有个初始值
y 输出（Output） 变换后的另一串数字。维度和 x 可以不同 加工好的产品
💡 一句话总结
nn.Linear 就是一个"数字变换器"——你把一串数字交给他，他在内部做一次矩阵乘法（Wx），再加一个偏移（b），还你另一串数字。

这和 PS 里的"色相/饱和度"滑块很像：输入是图片的像素数据，经过固定的数学变换，输出是调整后的图片。不同的是——nn.Linear 的变换规则（W 和 b）不是人调的，而是模型自己从数据里学出来的。

用小数字走一遍，马上就有感觉了

别一上来就想 768 维——那只会吓到自己。假设输入只有 3 个数： 

x = [2, 3, 1]    ← 一个 3 维向量（3 个数排成一行）

我想把它变成一个 2 维向量（2 个数），怎么做？需要一个 2 行 × 3 列 的矩阵 W： 

     ┌           ┐
W =  │ 0.5  -1   0 │    ← 2 行（输出维度）× 3 列（输入维度）
     │ 0.2  0.7  1 │
     └           ┘

第一步：Wx —— 矩阵乘法（上一节刚学过！）：

Wx = [0.5×2 + (-1)×3 + 0×1,   0.2×2 + 0.7×3 + 1×1]
   = [1.0 + (-3) + 0,   0.4 + 2.1 + 1]
   = [-2.0,  3.5]

第二步：加上偏置 b。假设 b = [1.0, -1.0]：

y = Wx + b = [-2.0 + 1.0,   3.5 + (-1.0)] = [-1.0, 2.5]

完事。输入 [2, 3, 1]（3 个数）→ 输出 [-1.0, 2.5]（2 个数）。 这就是 nn.Linear(3, 2) 做的一次完整运算。

⚠️ 关键观察
输入是 3 个数，输出是 2 个数。数量变了！这说明 nn.Linear 可以改变向量的维度——可以把 768 扩到 3072，也可以把 3072 压回 768。
正是这个"变维"能力，让 Linear 成了神经网络里最通用的积木。

in_features 和 out_features —— 就两个数字，别怕

现在回头看 nn.Linear(768, 3072)，这两个数字就是：

  • 第一个数 in_features=768：输入的向量有多少个数（"入口宽度"）
  • 第二个数 out_features=3072：输出的向量有多少个数（"出口宽度"）

画成图就是一根管子，一头细一头粗：

[768 个数]  →  ╔══════════════╗  →  [3072 个数]
  输入           ║  nn.Linear   ║       输出
                 ║  (768, 3072) ║
                 ╚══════════════╝

管子里面发生了什么？就是 Wx + b。而且 PyTorch 自动帮你把 W 和 b 建好了：

layer = nn.Linear(768, 3072)定义一个线性层：768 进，3072 出
# PyTorch 在背后自动创建：你不需要手动创建 W 和 b
# layer.weight → shape (3072, 768)W 矩阵：3072 行 × 768 列
# layer.bias → shape (3072,)b 向量：3072 个数
output = layer(input)一行搞定：等价于 input @ W.T + b
🔑 记住这个对应关系
nn.Linear(in, out) → W 的 shape 是 (out, in)。
输出维度在前，输入维度在后，和括号里的参数顺序一样。

GELU 是什么？—— 给直线加一点"弯"

现在你知道了 Linear 做的是 Wx + b，一个纯粹的线性运算——输入和输出之间是直线关系。

但这里有一个问题：如果你把 10 层 Linear 叠在一起，等效于 1 层 Linear。 因为线性叠线性还是线性——就像把 10 个放大镜串起来，等效于 1 个放大镜。叠再多也没用。

解决办法：在两层 Linear 之间插入一个"掰弯"操作。 把直线掰弯之后，多层叠加就不再等效于一层了——网络可以表达更复杂的模式。 这个"掰弯"操作就叫激活函数（Activation Function）。

GELU（Gaussian Error Linear Unit，高斯误差线性单元）是 Transformer 里最常用的激活函数。它做的事非常简单：

正数几乎不变，负数压到接近 0。 
GELU( 3.0) ≈  3.0    ← 大的正数 → 几乎不变
GELU( 0.5) ≈  0.35   ← 小的正数 → 缩小一些
GELU( 0.0) =  0.0    ← 零 → 还是零
GELU(-0.5) ≈ -0.15   ← 负数 → 被压到接近 0
GELU(-3.0) ≈ -0.01   ← 大的负数 → 几乎变成 0

你可以把 GELU 想象成一个"选择性通过"的阀门：正信号放行，负信号截断。 这种"选择性"引入了非线性——多层叠加不再等效于一层了。

⚠️ 初学者不用深究 GELU 的数学公式。
你只需要知道三点：
① 它是"激活函数"——给网络加入非线性；
② 效果是"正数通过、负数压到接近 0"；
③ 在代码里就是一行 F.gelu(x)，不需要自己实现。

拼起来：FFN = 两个 Linear 夹一个 GELU

现在三个零件你都认识了：

  1. nn.Linear = Wx + b（线性变换，改变向量维度）
  2. GELU = 激活函数（加入非线性，让多层叠加有意义）
  3. 矩阵乘法（上一节学的，Linear 内部用的就是它）

Transformer 里的前馈网络（FFN, Feed-Forward Network）就是这三个零件的简单组合。 它的公式：

输出 = W₂ × GELU(W₁ × 输入 + b₁) + b₂

下面逐段标注，把每个部分和 nn.Linear 对应上：

输出 =  W₂   ×  GELU(  W₁ × 输入 + b₁  )  +  b₂
       ↑          ↑      └───────────────┘       ↑
       │          │         这是第一个            │
       │          │       nn.Linear(768, 3072)   │
       │          │         = W₁x + b₁            │
       │          │                               │
       │          └────────── GELU ──────────────┘
       │                    激活函数，加非线性
       │
       └────────── 第二个 nn.Linear(3072, 768) ──┘
                       = W₂x + b₂

翻译成 PyTorch 代码，三行搞定：

# 定义两个 Linear 层（fc = fully connected = 全连接，Linear 的别名）
fc1 = nn.Linear(768, 3072)      # W₁x + b₁，768 维 → 3072 维
fc2 = nn.Linear(3072, 768)      # W₂x + b₂，3072 维 → 768 维

# 一次完整的前向计算（输入 → FFN → 输出）
x = fc1(input)       # 第一步：Linear，扩到 4 倍
x = F.gelu(x)        # 第二步：GELU 激活，加入非线性
output = fc2(x)      # 第三步：Linear，压回原尺寸

数据流动的完整路径：

768 维 ──→ [fc1: Linear(768,3072)] ──→ 3072 维 ──→ [GELU] ──→ 3072 维 ──→ [fc2: Linear(3072,768)] ──→ 768 维
          └━━━ 扩张 4 倍 ━━━━┘          └━ 加非线性 ━┘          └━━━ 压缩回来 ━━━━┘
🔗 回到 L5：这就是 Transformer Block 里的 FFN！
先扩到 4 倍（768→3072），给模型一个"更宽的思考空间"；
在中间用 GELU 做非线性变换，让多层叠加有意义；
再压回原来的尺寸（3072→768），以便和下一层对接。
这个"扩张 → 非线性 → 压缩"的结构，是每一个 Transformer Block 的标准配置。

概念四：nn.Embedding —— 这就是第 3 课的查表

回到 L3，你理解了 Embedding 的本质：每个 token ID 对应一个向量，Embedding 层就是一张巨大的查找表。 在 PyTorch 里，它就是 nn.Embedding(num_embeddings, embedding_dim)：

embed = nn.Embedding(50257, 768)定义：50257 个词 × 每个 768 维向量
# embed.weight shape=(50257, 768)这就是那张「大表」—— 50257 行，每行 768 个数
token_ids = torch.tensor([42, 318, 7])假设这三个 token ID 代表 "我 喜欢 猫"
vecs = embed(token_ids)查表：取出第 42、318、7 行的向量
# vecs.shape → (3, 768)3 个 token，每个变成 768 维向量

Embedding 查表nn.Embedding(vocab, dim)(vocab, dim)
Token ID → 向量embed(token_ids)(seq_len, dim)

概念五：autograd —— 自动求导，不用手算梯度了

在 L6 你理解了梯度下降：算 loss → 求每个参数的梯度 → 沿梯度反方向更新参数。 但当时说的是「手算导数」。现实中模型有上亿参数，手算是不可能的。

autograd 就是 PyTorch 的自动求导引擎。 它的工作原理很简单：

  1. Forward（前向）：正常做计算，PyTorch 在后台悄悄记录每一步操作，构建一张「计算图」
  2. Backward（反向）：调用 loss.backward()，PyTorch 从 loss 出发，沿着计算图往回走，自动算出每个参数的梯度
  3. 梯度存在 param.grad 里，直接用就行
x = torch.tensor([2.0], requires_grad=True)告诉 PyTorch：「这个变量我需要求梯度」
y = x ** 2 + 3 * x + 1y = x² + 3x + 1，PyTorch 自动记录计算过程
y.backward()自动计算 dy/dx，结果存在 x.grad 里
x.grad → tensor([7.])dy/dx = 2x + 3 = 2×2 + 3 = 7 ✅ 不需要你手算

🔬 动手试试：微型 autograd 模拟器

这是一个极简计算图：y = w × x + b（就像 nn.Linear 的一个神经元的计算！）

🔗 回到 L6 训练循环 有了 autograd，整个训练循环简化成了五步：
output = model(input) → loss = criterion(output, target) → loss.backward() → optimizer.step() → optimizer.zero_grad()
其中 backward() 是 autograd 的入口，之前 L6 讲的「反向传播求梯度」就是它自动完成的。

把所有零件拼起来：一个微型前向传播

下面这段代码用了你刚学的所有 PyTorch 操作。读一遍，看注释——它和 Phase 1 的架构一一对应：

import torch
import torch.nn as nn

# --- 1. Embedding：对应 L3 ---
vocab_size = 1000         # 假设词表有 1000 个词
d_model = 64              # 每个词的向量维度
embed = nn.Embedding(vocab_size, d_model)

# --- 2. Linear：对应 L5 的 FFN ---
ffn = nn.Linear(d_model, 4 * d_model)  # 768 → 3072 的缩小版：64 → 256

# --- 3. 模拟输入 ---
input_ids = torch.tensor([[1, 5, 8, 2]])  # 一个句子，4 个 token
print(f"输入 shape: {input_ids.shape}")    # (1, 4)  → 1 个句子，4 个 token

# --- 4. 查 Embedding 表 ---
x = embed(input_ids)
print(f"Embedding 后 shape: {x.shape}")    # (1, 4, 64)  → 1 句 × 4 词 × 64 维

# --- 5. 过 FFN ---
out = ffn(x)
print(f"FFN 后 shape: {out.shape}")        # (1, 4, 256) → 每个词从 64 维扩到了 256 维

# --- 6. 每个参数的形状 ---
for name, param in embed.named_parameters():
    print(f"embed.{name} shape: {param.shape}")    # (1000, 64) — 大查找表

for name, param in ffn.named_parameters():
    print(f"ffn.{name} shape: {param.shape}")      # weight (256, 64), bias (256,)
