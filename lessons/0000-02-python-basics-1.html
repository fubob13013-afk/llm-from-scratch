1 变量 — 给数据起个名字

变量就是一个有名字的盒子，里面装着一个值。

x = 5               ← 把数字 5 放进叫 "x" 的盒子
name = "Bob"        ← 把文字 "Bob" 放进叫 "name" 的盒子
pi = 3.14159        ← 把小数放进去

Python 里的 =（等号）不是「等于」，而是「赋值」——把右边的值装进左边的盒子。

三种最基本的「值」的类型：

类型Python 里的叫法例子能做什么
整数int42, -7, 0加减乘除
小数float3.14, -0.5, 1e-4科学计算
文字串str（string）"你好", 'world'拼接、查找、切片
Python 游乐场 — 试试修改变量的值
🔑 关键规则 Python 里变量名不用提前声明类型。写 x = 42，Python 自动知道 x 是整数。写 x = "hello"，同一个 x 又能变成字符串。这叫动态类型——类型跟着值走，不跟着变量名走。

2 列表 — 把很多东西装在一起

列表（List）就是一个有顺序的容器，可以装任意多个东西。

fruits = ["苹果", "香蕉", "橘子", "葡萄"]    ← 一个列表，4 个元素
numbers = [1, 3, 5, 7, 9]                    ← 全是数字的列表
mixed = ["Bob", 20, 3.14, True]              ← 混合类型也行

⚠️ 初学者最容易犯的错：索引从 0 开始

3 for 循环 — 让计算机重复做事

如果说变量是「盒子」，循环就是「流水线」——对列表里的每一个元素，依次做同样的事。

tokens = ["我", "喜欢", "AI"]

for token in tokens:          ← 逐个取出列表里的元素
    print(token)              ← 对每个元素执行这行（注意前面有 4 个空格缩进）

# 输出：
# 我
# 喜欢
# AI

拆解这个结构：

  • for ... in ...: = 「对于……里的每一个……，做以下事情」
  • 冒号后面的缩进（4 个空格或 Tab） = 「以下操作属于这个循环」
  • 不再缩进的行 = 「循环结束了，继续往下执行」

range() — 重复 N 次

range(5) 生成 0, 1, 2, 3, 4 这 5 个数字。配合 for 循环就是「重复 5 次」：

for i in range(5):           ← i 依次 = 0, 1, 2, 3, 4
    print("第", i+1, "次")    ← i+1 让显示从 1 开始
# 输出：第 1 次 / 第 2 次 / 第 3 次 / 第 4 次 / 第 5 次

4 if 判断 — 让计算机做选择

if 语句就是「如果……就……否则……」：

score = 85

if score >= 60:               ← 如果分数 ≥ 60
    print("及格")              ← 就执行这行
else:                         ← 否则（分数 < 60）
    print("不及格")            ← 执行这行

# 输出：及格

常用比较符号：

==等于（注意是双等号！单等号是赋值）
!=不等于

🧪 小测验

问题 1/3 — 索引

data = ["A", "B", "C", "D"]。请问 data[1] 和 data[-1] 分别是什么？

点击看答案
data[1] = "B"（第二个元素）  |  data[-1] = "D"（最后一个元素）。
索引从 0 开始：0→A, 1→B, 2→C, 3→D。负数从末尾倒数：-1→D, -2→C。

问题 2/3 — for 循环

下面的代码会打印几个数字？分别是多少？

for i in range(3):
    print(i)
点击看答案
3 个数字：0, 1, 2。
range(3) 生成 0, 1, 2（从 0 开始，共 3 个，不包含 3）。如果要从 1 开始，用 range(1, 4)。

问题 3/3 — if 判断

以下代码会输出什么？

token = "[PAD]"
if token == "[PAD]":
    print("跳过")
else:
    print("保留")
点击看答案
输出「跳过」。
token == "[PAD]" 判断 token 的值是否等于字符串 "[PAD]"。因为确实是，所以执行 if 分支。
注意 ==（判断）不是 =（赋值）。如果写成 if token = "[PAD]":——报错！
