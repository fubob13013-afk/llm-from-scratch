§1 完整 GPT 架构 —— 拼好所有拼图

现在可以画出完整的前向传播流程图了：

输入 Token ID 序列：[15, 27, 8, 42, …, 91]   (如 1024 个 token)
              │
              ▼
    ┌─────────────────────────┐
    │  Token Embedding        │  查表：每个 ID → 768 维语义向量
    │  + Position Encoding    │  + 768 维位置指纹向量（第 3 课）
    └─────────────────────────┘
              │
              ▼
    ┌─────────────────────────┐
    │     Dropout             │  随机关掉 10% 的神经元，防止过拟合
    └─────────────────────────┘
              │
              ▼
    ╔═════════════════════════╗
    ║  Transformer            ║
    ║  Block 1                ║  LN → Multi-Head Attention → +x
    ║  Block 2                ║  LN → FFN → +x          （第 5 课）
    ║  …                     ║  参数不同，结构相同
    ║  Block 12               ║
    ╚═════════════════════════╝
              │
              ▼
    ┌─────────────────────────┐
    │  Final LayerNorm        │  最后一次归一化
    └─────────────────────────┘
              │
              ▼
    ┌─────────────────────────┐
    │  Linear + Softmax       │  768 维 → 词表大小（50000 维）
    │                         │  Softmax 转成概率分布
    └─────────────────────────┘
              │
              ▼
    输出：每个位置上，下一个 token 的概率分布
    [P(北京|…), P(上海|…), P(的|…), …]  (50000 个概率)

写成伪代码

def gpt_forward(token_ids):
    # 1. 嵌入（第 3 课）
    x = token_embedding(token_ids)      # [batch, 1024, 768]
    x = x + position_encoding(x)        # + 位置指纹

    # 2. 过 12 层 Transformer Block（第 5 课）
    for block in transformer_blocks:    # 12 个 Block，参数各自独立
        x = block(x)

    # 3. 输出
    x = layer_norm_final(x)
    logits = linear(x)                  # [batch, 1024, 50000]
    probs = softmax(logits)             # 转成概率

    return probs  # 每个 token 位置预测下一个 token

这就是 GPT-2 小型版（124M 参数）的完整计算流程。大模型（GPT-3、GPT-4）的区别只在：更多层（96 层）、更宽（12288 维）、更多数据——结构完全一样。

§2 文本生成 —— 一个 token 一个 token 地产出

自回归解码（Autoregressive Decoding）

模型一次只能预测一个 token。要生成一整段文本，需要把刚才生成的 token 接在输入后面，再喂给模型预测下一个。如此循环：

第 1 步：输入「中国的首都是」→ 模型预测「北」
第 2 步：输入「中国的首都是北」→ 模型预测「京」
第 3 步：输入「中国的首都是北京」→ 模型预测「。」
第 4 步：输入「中国的首都是北京。」→ 模型预测 <EOS>（结束符）
                                    → 生成完成

每次预测时，模型输出的是整个词表（50000 个 token）上的概率分布。怎么从概率分布中选出一个 token？

三种采样策略

策略做法效果
贪心（Greedy） 每次都选概率最高的 token 确定性强，但容易陷入重复——模型会一直选最「安全」的词
Temperature 采样 把概率除以 temperature，再按比例随机选 T→0 → 接近贪心；T=1 → 自然随机；T→∞ → 完全随机
Top-k / Top-p 只从概率最高的 k 个中选，或从累积概率达到 p 的最小集合中选 过滤低概率噪声，在质量和多样性间取平衡

Temperature 在做什么？

模型先算出每个 token 的原始分数（logits），用 softmax 转成概率。Temperature 在 softmax 之前把 logits 除以 T：

概率 = softmax( logits / T )
  • T < 1（如 0.5）：logits 变大 → 高分 token 概率更高，低分 token 概率更低 → 输出更「确定」
  • T = 1：不改动，按原始概率采样
  • T > 1（如 1.5）：logits 缩小 → 概率分布更均匀 → 输出更「随机」

把一切串起来：7 课完整旅程

  1. L1 · LLM 核心任务：预测下一个 token，自监督学习
  2. L2 · 数据准备：原始文本 → 分词(BPE) → Token ID → 滑动窗口 → DataLoader
  3. L3 · 词嵌入与位置编码：Token ID → 768 维语义向量 + 位置指纹
  4. L4 · 注意力机制：Q·K 匹配 → Softmax → 加权聚合 V，因果遮罩
  5. L5 · Transformer Block：多头注意力 + FFN + 残差 + LayerNorm，重复 12 次
  6. L6 · 训练循环：前向 → 损失 → 反向 → 更新，反复百万次
  7. L7 · 组装与生成（本课）：完整架构串联，自回归逐 token 生成文本

一个 GPT 不是魔法——它就是一个结构固定的数学函数（Embedding + 12 层 Block + 输出投影），加上反复训练出来的几千亿个参数，再加上自回归的文本生成策略。
