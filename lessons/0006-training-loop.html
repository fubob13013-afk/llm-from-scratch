🔄 第6课 · 训练循环 —— 模型怎么学？

前 5 课讲完了「模型长什么样」——Embedding、Attention、多头、FFN、残差、LayerNorm。本课回答：这些部件里的几十亿个参数，是怎么被调到合适的值的？

⚠️ 本课前提 需要知道 Transformer Block 的结构（第 5 课），特别是残差连接中提到的「梯度」概念。本课把梯度、损失函数、训练循环完整展开。

回顾：目前我们有什么？

一串文本经过以下流水线：

原始文本
→
分词
Token ID
→
Embedding
+ 位置编码
→
×12 Block
Attention + FFN + 残差 + LN
→
输出
下一个 token 的概率

这个流水线里塞满了参数——W^Q, W^K, W^V, W^O, W₁, W₂, b₁, b₂, γ, β……总共几十亿个数字，一开始全是随机数。如果就这样拿去预测下一个 token，结果基本是瞎猜。

训练的过程，就是反复做一件事：让模型猜 → 看错得多离谱 → 算出每个参数该往哪边调 → 微调 → 再来一次。

§1 损失函数（Loss Function）—— 用数字衡量「错得多离谱」

为什么需要一个数字？

上面说的「看错得多离谱」，需要一个客观的、可比较的数字。这个数字就是损失值（Loss）。

LLM 的训练目标是预测下一个 token。所以损失函数问的问题是：

模型给「正确答案」的概率有多高？概率越高 → 损失越低。

交叉熵损失（Cross-Entropy Loss）

LLM 使用的损失函数叫交叉熵损失：

Loss = -log( 模型给正确 token 的概率 )
模型给正确答案的概率Loss意味着
0.9（90%）-ln(0.9) ≈ 0.105模型很确定，loss 很低
0.5（50%）-ln(0.5) ≈ 0.693模型在猜硬币，一半把握
0.1（10%）-ln(0.1) ≈ 2.30模型基本没往正确方向想
0.00002（1/50000）-ln(0.00002) ≈ 10.8完全瞎猜（词表均匀分布）

为什么用 log？因为概率从 0.9 掉到 0.1，在人的直觉里是「差了一点」，但 -ln 把差距拉大了（0.105 → 2.30），让模型对「不靠谱的预测」受到更大的惩罚。

§2 梯度下降 —— 沿着山坡往下走

有了损失值，就知道了「当前有多差」。下一步：往哪个方向调参数能让损失变小？

导数 = 指路的向导

损失 L 是所有参数的函数：L = f(W₁, W₂, b₁, …)。对某个参数求导，得到的是：

「其他参数都不动，单独把这个值调大一丁点，损失会怎么变？」

  • 导数 > 0 → 调大它会让损失变大 → 减小它
  • 导数 < 0 → 调大它会让损失变小 → 增大它

更新规则：新值 = 旧值 - 学习率 × 导数

学习率（Learning Rate）—— 步子迈多大

  • 太大：在最低点附近反复横跳，甚至越跳越高
  • 太小：走得太慢，训练到天荒地老还没到谷底
  • 刚好：稳步下降，收敛到最低点

§3 一个完整的训练步骤（Training Step）

① 前向传播
Forward Pass
→
② 计算损失
Compute Loss
→
③ 反向传播
Backward Pass
→
④ 更新参数
Update
→ 回到 ①

① 前向传播（Forward Pass）

把输入数据喂给模型，一路算到输出。此时参数不动——Embedding → Attention → FFN → … → 输出概率分布。

② 计算损失（Compute Loss）

把预测结果和正确答案对比，用交叉熵算一个数字。

③ 反向传播（Backward Pass）

从损失出发，用链式法则逐层往回算每个参数的梯度。这一步不更新参数，只是算出「每个参数该往哪边调」。

④ 更新参数（Parameter Update）

每个参数按照 新值 = 旧值 - 学习率 × 梯度 微调。几十亿个参数同时做这件事。

然后回到第 ① 步，用下一批数据再来一遍。反复几百万次。

§4 怎么知道「学好了」？—— 训练损失 vs 验证损失

训练损失（Training Loss）
在训练数据上计算的损失。这个值用于反向传播，直接驱动参数更新。
验证损失（Validation Loss）
在模型没见过的数据上计算的损失。这个值不用于更新参数，纯粹用来观察——模型是真的学会了规律，还是在背训练集？

过拟合（Overfitting）

如果训练损失持续下降而验证损失开始上升，说明模型在「背答案」而不是「学规律」：

训练损失：↘↘↘↘↘↘↘ （一直在降，模型在努力背）
验证损失：↘↘↘↗↗↗ （先降后升，背的东西在新题上不管用了）

实操中，训练到验证损失不再下降就停。这个策略叫早停法（Early Stopping）。

把一切串起来

到这里，你知道了：

  • 模型怎么算（第 2-5 课）：分词 → Embedding + 位置 → Attention → 多头 + FFN + 残差 + LN，12 层串联
  • 模型怎么学（本课）：重复「前向 → 损失 → 反向 → 更新」，每一步都让损失降一点点，几百万步后收敛

LLM 的训练就是这个循环的超级放大版：不是 2 个参数而是几千亿个，不是 50 个数据点而是几万亿个 token，不是单机 CPU 而是几千张 GPU 跑几个月。但核心逻辑一模一样。
