🤖 第1课 · LLM 做什么？—— 预测下一个词

这是整个课程的第一课。我们不碰任何公式，不做任何计算。只建立一件事：LLM 到底在干什么？

答案是四个字：猜下一个词。

一个实验：你来当 LLM

读下面这句话，猜横线上最可能是什么词：

中国的首都是_____

你脑子里大概率蹦出了「北京」。换一个：

他打开冰箱，拿出一瓶冰镇的_____

「可乐」「啤酒」「矿泉水」——你的大脑根据前半句的语境，从你毕生所学的中文知识中，选出了概率最高的几个候选词。你刚刚就做了一次语言模型推理。

LLM 做的事和你刚才做的一模一样——只不过：

  • 它读过的文本比你多几百万倍（整个互联网的文本）
  • 它做这个选择不是靠「直觉」，而是靠数学计算
  • 它每秒钟能做几万次这样的预测

给这个任务一个正式名字

在人工智能领域，这个任务叫：

给定前面的所有词，预测下一个词的概率分布。

拆开来看：

「前面的所有词」= 上下文（Context）
就是模型能看到的已输入的文本。GPT-4 的上下文窗口最多可以看 128,000 个 token——差不多一本小说的长度。
「下一个词」= 目标（Target）
模型要猜的那个词。在训练数据中这个答案是已知的——这就构成了「输入→正确答案」的训练对。
「概率分布」
模型不会只输出一个词。它输出的是每个可能词的得分——「北京 92%，上海 4%，广州 2%，……」。这个得分表就是概率分布。
🔑 核心心智模型 把 LLM 想象成一个超级输入法。你手机上的输入法在你打了「我今天想去」之后，会推荐「吃饭/逛街/睡觉/上班」。LLM 做的是同一件事——只是它看的不是前 10 个字，而是前 10 万个 token；它的候选词不是 50 个，而是 5 万个。

数据自己就是「标准答案」—— 自监督学习

训练 LLM 不需要雇人标注数据。这是它最精妙的地方。

假设你有这段文本：

人工智能正在改变世界

训练样本是自动生成的——把每个位置变成一道「填空题」：

输入（模型看到的）正确答案（模型要猜的）
_____人工
人工智能
人工 智能正在
人工 智能 正在改变
人工 智能 正在 改变世界

每道填空题的答案已经在原文里了。把整个互联网的文本都做成这样的填空题，就是几万亿道练习题。这种「数据自带标签」的学习方式叫自监督学习（Self-Supervised Learning）。

🔑 关键洞察 LLM 训练不需要人工标注。你用百度百科、知乎、GitHub、小红书帖子训练它——这些文本的作者在写的时候，已经无意中给每个位置填上了「标准答案」。模型要做的就是在每个位置猜，猜错了就根据差距调整自己的参数。

注意观察：每个位置的输入，正好是上一个位置的输入 + 上一个位置的目标。这就是为什么不需要人工标注——文本自己就是一条连续的链条，每个词都在为前面的词提供「标准答案」。

从「猜词」到「对话」—— 一个自然的结果

你可能会问：如果 LLM 只是一个「超级输入法」，它怎么做到回答问题、写诗、写代码这些看起来需要「理解」的事情？

答案令人意外：它不需要理解。它只需要把「对话」也当成一种文本模式来续写。

训练好的模型看到：

用户：法国的首都是哪里？
助手：

它续写出「巴黎」——不是因为它知道地理，而是因为在几万亿字的训练数据中，这段文字后面最常出现的那个词就是「巴黎」。

这并不是说 LLM 「不懂」——当预测下一个词的准确率高到一定程度时，它表现出的行为和「理解」难以区分。这就像一个乒乓球运动员不需要懂物理也能打出完美的弧圈球——他的神经网络（大脑 + 肌肉）已经内化了物理规律。

延伸：LLM 的完整生命周期（选读）

本课聚焦在「预测下一个词」这个核心任务上。但一个商业可用的 LLM（比如 ChatGPT、文心一言），从零到成品还要经历更多阶段。这里简要列出，建立一个「课程地图」——知道我们现在在学的是哪一块：

阶段做什么我们的课程覆盖？
预训练 用海量文本训练「预测下一个词」的能力 ✅ 全部 7 课都在讲这个
微调 用几千条高质量对话教模型「聊天格式」 以后拓展
对齐 用人类反馈让模型输出安全、有帮助的内容 以后拓展
部署 压缩模型 + 包装成 API 或 App 以后拓展

预训练是最核心、最烧钱、也最体现技术含量的阶段。这 7 节课学的就是预训练——你理解了这个，就理解了 LLM 80% 的技术本质。

🧪 小测验

问题 1/3 — 核心任务

LLM 的核心任务是什么？用一句话回答。

点击看答案
给定上文，预测下一个词（token）的概率分布。
换个说法：LLM 就是一个超级输入法——根据你已输入的内容，推荐接下来最可能出现的词。

问题 2/3 — 自监督学习

训练 LLM 为什么不需要人工标注数据？

点击看答案
因为文本自己就包含了「标准答案」。对于句子「A B C D」，训练样本自动生成为：看到「A」猜「B」、看到「A B」猜「C」、看到「A B C」猜「D」。每个位置的正确答案已经在原文中了。这叫自监督学习。

问题 3/3 — 理解 vs 预测

一个只会「猜下一个词」的模型，为什么能回答问题、写代码、翻译语言？

点击看答案
因为在这些任务中，「正确的下一个词」恰好就是需要理解才能选对的。
比如「法国的首都是___」，正确的下一个词就是「巴黎」。模型不需要真的「知道」地理——它只需要从训练数据中学到：在这句话后面，「巴黎」出现的概率远大于其他词。当预测准确率高到一定程度，表现出来的就是「理解」。
