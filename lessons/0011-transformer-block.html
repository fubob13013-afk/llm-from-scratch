🧱 第11课 · 实现 Transformer Block

Block：12 层积木，每层一模一样

L10 实现了注意力，现在要把它和 FFN、残差连接、LayerNorm 打包成一个标准的Transformer Block。 GPT-2 有 12 个完全一样的 Block 叠在一起——数据从第 1 层流到第 12 层，每一层都在上一层的输出上做进一步的理解。

📐 一个 Transformer Block 的结构（Pre-LN 顺序）：
输入 x → LN → Attention → +x → LN → FFN → +x → 输出

拆开来看就四个零件，每个你都见过：

零件代码作用哪学的？
LayerNormnn.LayerNorm(768)把数据压到均值为 0、方差为 1L5
Multi-Head AttentionMultiHeadAttention()让 token 之间互相「看」L10
FFNLinear→GELU→Linear对每个 token 独立做非线性变换L5、L8
残差连接x = x + sublayer(x)让梯度直接穿透到深层L5

概念一：LayerNorm —— 给数据「梳理」一下

L5 讲了 LayerNorm 的原理：把每个 token 的向量标准化，均值变为 0，方差变为 1，然后乘以可学习的 γ 加上 β。 在 PyTorch 里，它就是一行：

self.ln1 = nn.LayerNorm(d_model)定义：对 768 维向量做归一化
x_norm = self.ln1(x)使用：(B, L, 768) → (B, L, 768)，shape 不变

为什么 Attention 和 FFN 之前都要先过 LayerNorm？因为经过多层处理后， 数值的分布会逐渐漂移（有的维度变得很大，有的变得很小）—— 这会破坏训练稳定性。LayerNorm 在每一个 Block 的入口和中间各做一次「梳理」，确保数值始终保持健康分布。

💡 LayerNorm vs BatchNorm
BatchNorm 对同一个 batch 里的不同样本做归一化（跨 batch 维度）——这在 NLP 里不稳定，因为句子长度不一。
LayerNorm 对每个样本自己做归一化（跨特征维度）——与 batch 大小无关，训练和推理行为一致。
这就是为什么 Transformer 选 LayerNorm 而不是 BatchNorm。

概念二：FFN —— 每个 token 的「独立思想」

注意力的作用是「token 之间交换信息」。但交换完之后，每个 token 需要独立地「消化」这些信息—— 这就是 FFN（前馈网络）的角色。

FFN 的结构极其简单——两个 Linear 夹一个 GELU 激活函数：

self.ffn = nn.Sequential(Sequential：把多层打包成一个模块
nn.Linear(768, 3072),第一步：从 768 维扩张到 3072 维（4 倍）
nn.GELU(),第二步：非线性激活函数（比 ReLU 更平滑）
nn.Linear(3072, 768),第三步：从 3072 维压缩回 768 维
)输入输出 shape 相同：(B, L, 768) → (B, L, 768)
🔗 回到 L5 「扩张到 4 倍再压缩」这个设计，L5 讲过——它给了模型一个高维空间来暂存中间计算结果， 类似于草稿纸上写演算过程，最终只保留结论。 GELU（Gaussian Error Linear Unit）相比 ReLU 更平滑，梯度流动更顺畅。

概念三：残差连接 —— 让梯度「走高速」

GPT-2 有 12 层 Block。如果没有残差连接，梯度从最后一层传回第一层时，会被连乘 12 次—— 数值会变得非常小（梯度消失），前面的层几乎学不到东西。

残差连接就是在每个子层（Attention/FFN）外面加一条「旁路」： 

# 不是：x = attention(x)    ← 替换 x
# 而是：x = x + attention(x) ← 在原有 x 基础上加上 attention 的贡献

后果很神奇：反向传播时，梯度可以通过旁路直达浅层，不被 12 层变换削弱。 这是让深层网络可训练的核心技巧。

💡 直觉理解
把残差连接想象成一条「高速公路」：正常情况下车要从 A 到 B 需要绕山路（经过 Attention/FFN 的计算）， 但残差连接在 A 和 B 之间直接修了一条隧道——信息可以直通。训练时，梯度也可以通过这条隧道直达浅层，避免衰减。

概念四：Pre-LN —— LN 先做，再交给子层

有两种顺序：

顺序结构特点
Post-LN（原始 Transformer）Attention → +x → LN不稳定，容易梯度爆炸
Pre-LN（GPT-2 用的）LN → Attention → +x稳定，梯度更平滑

GPT-2 选择了 Pre-LN：输入先进 LayerNorm 做标准化，再交给 Attention/FFN 处理，最后加上残差。 这里的「Pre」指的是 LN 在子层之前。

完整代码：TransformerBlock 类

import torch
import torch.nn as nn

class TransformerBlock(nn.Module):
    """GPT-2 的一个 Transformer Block（12 个这样的 Block 叠起来）"""

    def __init__(self, d_model=768, num_heads=12, dropout=0.1):
        super().__init__()

        # --- 子层 1：自注意力 ---
        self.ln1 = nn.LayerNorm(d_model)          # Pre-LN：先归一化
        self.attn = MultiHeadAttention(           # 来自 L10
            d_model=d_model,
            num_heads=num_heads,
            dropout=dropout
        )

        # --- 子层 2：前馈网络 ---
        self.ln2 = nn.LayerNorm(d_model)          # Pre-LN：先归一化
        self.ffn = nn.Sequential(
            nn.Linear(d_model, 4 * d_model),      # 768 → 3072（4倍扩张）
            nn.GELU(),                            # 非线性激活
            nn.Linear(4 * d_model, d_model),      # 3072 → 768（压缩回来）
        )

        self.dropout = nn.Dropout(dropout)

    def forward(self, x):
        # --- 子层 1：Attention（Pre-LN） ---
        # 注意这里只有两行，但就是完整的注意力子层！
        normed = self.ln1(x)                # 先 LN
        x = x + self.dropout(self.attn(normed))  # 再 Attention + 残差

        # --- 子层 2：FFN（Pre-LN） ---
        # 完全相同的模式：LN → 子层 → +残差
        normed = self.ln2(x)                # 先 LN
        x = x + self.dropout(self.ffn(normed))   # 再 FFN + 残差

        return x


# ===== 测试：12 个 Block 叠起来 =====
from L10_attention import MultiHeadAttention  # 假设 L10 的代码在此

class MiniGPT(nn.Module):
    def __init__(self, num_layers=12):
        super().__init__()
        self.blocks = nn.Sequential(*[
            TransformerBlock() for _ in range(num_layers)
        ])

    def forward(self, x):
        return self.blocks(x)

model = MiniGPT(num_layers=12)
x = torch.randn(2, 512, 768)     # 模拟输入
out = model(x)
print(f"输入: {x.shape}")         # (2, 512, 768)
print(f"输出: {out.shape}")       # (2, 512, 768) — 12 层后 shape 不变！
📊 关键洞察 12 层 Block 叠在一起，输入和输出的 shape 完全一样：(B, L, 768)。 这就是「残差连接 + Pre-LN」的威力——每一层都是增量修改，而不是完全替换。 这意味着：① 梯度可以无损穿过每一层 ② 可以在训练好的模型中间插入/删除层 ③ 某层「罢工」了输出也不会崩溃。
