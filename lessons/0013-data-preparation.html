概念一：选数据 —— 越小越好（一开始）

训练 GPT 不需要海量数据。对于学习目的，几百 KB 的文本就够了——模型会在小数据上快速过拟合，让你看到 loss 下降，获得正反馈。

推荐的数据来源（从小到大）：

数据大小适合
一篇小说（如《三体》选段）~100KB快速验证代码
维基百科中文子集~50MB正式小规模训练
开源中文语料（如 BaiduQA）~500MB得到更像样的结果

with open('data.txt', 'r', encoding='utf-8') as f:

概念二：分词 —— 文本变成数字

L2 讲过 BPE（字节对编码）的原理。这里直接用它——tiktoken 是 OpenAI 开源的分词器，和 GPT-2 完全一致。 

import tiktoken

# 加载 GPT-2 的分词器
tokenizer = tiktoken.get_encoding("gpt2")

# 分词
text = "我喜欢猫"
token_ids = tokenizer.encode(text)
print(token_ids)   # 例如: [57668, 12345, 23456, ...]

# 还原
decoded = tokenizer.decode(token_ids)
print(decoded)     # "我喜欢猫"

# 词表大小
print(tokenizer.n_vocab)  # 50257
⚠️ 中文注意 GPT-2 的 tiktoken 主要面向英文，中文 token 会被切得更碎（一个汉字可能 2-3 个 token）。 中文专用模型（如 ChatGLM、Qwen）有自己的分词器，但原理相同。本课程用 GPT-2 分词器是为了保持和模型架构一致。

概念三：滑动窗口 —— 创造训练样本

GPT 的训练任务是「给定前面的 token，预测下一个 token」。 我们需要从一长串 token ID 中切出输入-目标配对。

核心思路：用一个固定大小的窗口在 token 序列上滑动，每次滑动产出两个东西：

  • 输入（input）：窗口内的前 N-1 个 token（模型看到的）
  • 目标（target）：窗口内的后 N-1 个 token（模型要预测的 = 输入向右移 1 位）

对应的 PyTorch 代码：

def create_sliding_windows(token_ids, max_len, stride):max_len=窗口大小，stride=步长
inputs, targets = [], []
for i in range(0, len(token_ids) - max_len, stride):从 0 开始，每次跳 stride 步
inputs.append(token_ids[i : i + max_len])窗口内的前 max_len 个 token
targets.append(token_ids[i + 1 : i + max_len + 1])向右移 1 位：每个位置的「下一个 token」
return inputs, targets
💡 为什么 target 是 input 右移一位？
这就是「预测下一个 token」的数学表达——input[i] 的正确答案是 target[i]（即 input[i+1]）。 训练时模型看着 input，被要求输出 target。这叫做自监督学习——不需要人工标注，数据自己就是标签。

概念四：DataLoader —— 打包成 batch

滑动窗口产出了几千个小片段。训练时不能一个一个喂——太慢。 需要打包成 batch（一批同时处理 2-8 个片段），利用 GPU 的并行能力。

完整代码：数据准备流水线

import torch
from torch.utils.data import DataLoader, Dataset
import tiktoken

class GPTDataset(Dataset):
    def __init__(self, filepath, tokenizer_name="gpt2", max_len=256, stride=128):
        # 1. 读取文本
        with open(filepath, 'r', encoding='utf-8') as f:
            text = f.read()

        # 2. 分词 → token ID
        tokenizer = tiktoken.get_encoding(tokenizer_name)
        token_ids = tokenizer.encode(text)
        print(f"文本长度: {len(text):,} 字符 → {len(token_ids):,} tokens")

        # 3. 滑动窗口切分
        self.inputs, self.targets = [], []
        for i in range(0, len(token_ids) - max_len, stride):
            self.inputs.append(token_ids[i : i + max_len])
            self.targets.append(token_ids[i+1 : i + max_len + 1])

        print(f"生成 {len(self.inputs):,} 个训练样本")

    def __len__(self):
        return len(self.inputs)

    def __getitem__(self, idx):
        return torch.tensor(self.inputs[idx]), torch.tensor(self.targets[idx])


def create_dataloader(filepath, batch_size=4, max_len=256, stride=128):
    dataset = GPTDataset(filepath, max_len=max_len, stride=stride)
    dataloader = DataLoader(dataset, batch_size=batch_size,
                            shuffle=True, drop_last=True)
    return dataloader


# 使用示例
dataloader = create_dataloader("data.txt", batch_size=4)
for inputs, targets in dataloader:
    print(f"Batch: inputs {inputs.shape}, targets {targets.shape}")
    # inputs:  (4, 256)  — 模型输入
    # targets: (4, 256)  — 正确答案（用来算 loss）
    break
📊 数据要点总结
• 训练集/验证集划分：通常 9:1，验证集不参与训练，只用来评估
• shuffle=True：每个 epoch 打乱顺序，防止模型记住数据顺序
• drop_last=True：丢弃最后不完整的 batch，避免 shape 不一致
• stride < max_len：窗口之间有重叠，数据利用率更高
