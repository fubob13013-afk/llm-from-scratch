1 字典 — 给每个值起个名字

列表用数字索引（第 0 个、第 1 个……）。字典（dict）用名字索引：

# 列表：按位置取
info = ["Bob", 20, "清华美院"]
name = info[0]     # 要记住位置 0 是名字……

# 字典：按名字取
info = {"name": "Bob", "age": 20, "school": "清华美院"}
name = info["name"]   # 一目了然

字典的语法：{"键": 值, "键": 值, ...}。键（key）通常是字符串，值（value）可以是任何类型。

Python 游乐场 — 字典

这就是后续课程里的模型配置（config）——一个大字典，装满了各种参数设置。

2 函数 — 把一段代码打包

为什么需要函数？

如果你有一段代码需要在 100 个地方用到，你会复制粘贴 100 次吗？不会——你会给它起个名字，每次用到的时候叫它的名字就行。这就是函数。

❌ 没有函数：重复 3 次

# 第 1 次
result1 = 1 + 2 + 3 + 4 + 5
print(result1 / 5)

# 第 2 次
result2 = 6 + 7 + 8 + 9 + 10
print(result2 / 5)

# 第 3 次
result3 = 11 + 12 + 13 + 14 + 15
print(result3 / 5)

✅ 有函数：写一次，调用 3 次

def average(numbers):
    total = 0
    for n in numbers:
        total = total + n
    return total / len(numbers)

print(average([1,2,3,4,5]))
print(average([6,7,8,9,10]))
print(average([11,12,13,14,15]))

函数的四个要素

要素代码含义
定义def 函数名(参数):「我要定义一个函数，它叫这个名字，接收这些输入」
参数numbers函数的「输入口」——调用时把数据从这里传进去
返回return 值函数的「输出口」——算完的结果从这里传出去
调用average([1,2,3])「Hey average 函数，给你这组数据，返回结果给我」
Python 游乐场 — 函数
🔑 函数的关键心智模型 函数 = 一个加工机器。你把原材料（参数）从入口塞进去，机器内部处理，成品（返回值）从出口出来。

result = average([1,2,3,4,5])
→ 把 [1,2,3,4,5] 塞进 average 机器 → 机器算出 3.0 → 把 3.0 赋值给 result

后续课程中，model = GPTModel(config) 就是这个模式——把配置字典塞进 GPTModel 函数/类，返回一个构建好的模型。

3 import — 用别人写好的代码

你不需要从零写所有代码。Python 有海量的「包」（package）——别人写好、测试好、打包好的函数和工具，你直接拿来用。

import 就是「把那个包的代码加载进来」。

import torch                    ← 把 PyTorch 包加载进来

# 现在可以用 torch 里的东西了
x = torch.randn(3, 768)        ← 生成一个 3×768 的随机矩阵
layer = torch.nn.Linear(768, 768)  ← 创建一个线性层（就是第 5 课的 Wx+b）
y = layer(x)                   ← 前向传播：x 经过 layer 变换得到 y

三种 import 方式：

写法含义后续课程例子
import torch加载整个包，用 torch.xxx 访问torch.nn.Linear(...)
import torch.nn as nn加载子模块，起个短别名nn.Linear(...)（省打几个字）
from torch import nn只加载 nn 这一个子模块nn.Linear(...)

后续课程你会看到这些 import 语句出现在每个 Python 文件的最开头。它们是模型的「零件供应商」。

4 怎么看报错信息？—— 重要生存技能

写代码一定会遇到报错。不要怕报错——报错是 Python 在努力告诉你哪里出了问题。关键是学会读最后一行。

Python 游乐场 — 故意写 bug 看报错

💡 建议：逐个取消注释（删掉行首的 #），运行一次看报错信息，再改回来。这是学编程最快的方法——故意写 bug，看报错，理解为什么错了。

🔑 报错阅读三步法 ① 看最后一行：那行通常精确说明了错在哪（NameError? SyntaxError? IndexError?）
② 看行号：报错信息会告诉你是第几行出的问题（"line 5" 之类的）
③ 搜一下：把最后一行报错信息复制到百度/Bing 搜索，99% 的问题别人遇到过

学完了两节 Python 课的所有语法点。现在试着读下面这段代码——这就是 Phase 2 里训练 GPT 的核心逻辑。不需要完全理解每一行，只需要认出你学过的结构：

# Phase 2 课程中会出现的代码（现在先试着读一读）
import torch
import torch.nn as nn

# 配置字典
config = {
    "vocab_size": 50257,
    "emb_dim": 768,
    "n_layers": 12,
    "n_heads": 12,
    "max_length": 1024
}

# 定义模型（一个类，你可以理解为复杂的函数）
class GPTModel(nn.Module):
    def __init__(self, config):
        super().__init__()
        self.embedding = nn.Embedding(          # ← 第 3 课的查表
            config["vocab_size"], config["emb_dim"]
        )
        self.blocks = nn.ModuleList([           # ← 12 个 Transformer Block
            TransformerBlock(config)            # ← 第 5 课的结构
            for _ in range(config["n_layers"])  # ← for 循环！
        ])

    def forward(self, token_ids):              # ← 前向传播
        x = self.embedding(token_ids)           # ← 第 3 课
        for block in self.blocks:               # ← 第 5 课：逐层穿过
            x = block(x)                        # ← 第 5 课：每个 Block
        return x

# 训练循环
def train_one_step(model, batch, optimizer):    # ← 第 6 课
    inputs, targets = batch                     # ← 输入和目标（第 2 课）
    outputs = model(inputs)                     # ← 前向传播
    loss = compute_loss(outputs, targets)       # ← 计算损失（第 6 课）
    loss.backward()                             # ← 反向传播（第 6 课）
    optimizer.step()                            # ← 更新参数（第 6 课）
    return loss.item()                          # ← 返回损失值

🧪 小测验

问题 1/3 — 字典 vs 列表

什么时候用字典，什么时候用列表？给出一个例子。

点击看答案
当顺序重要、要逐个遍历时用列表；当需要用名字精确查找时用字典。
列表：tokens = ["我", "喜欢", "AI"] — 要按顺序逐个处理
字典：config = {"lr": 0.001, "batch_size": 64} — 要通过名字查具体的值

问题 2/3 — 函数的 return

下面这个函数在做什么？它的返回值是什么？

def compute_loss(prob):
    return -math.log(prob)
点击看答案
它计算交叉熵损失。接收一个概率值（0~1），返回 -ln(概率)。这是第 6 课讲的 Loss 函数。
return 后面的值就是函数的输出——调用 compute_loss(0.5) 会得到 0.693。

问题 3/3 — import 的作用

如果不写 import torch，直接写 x = torch.randn(3, 768)，会发生什么？

点击看答案
报错：NameError: name 'torch' is not defined。
Python 不认识 torch 是什么——因为还没有 import。import 相当于告诉 Python：「我要用 torch 这个工具箱，请加载它」。不 import，Python 就不知道 torch.randn 怎么执行。
