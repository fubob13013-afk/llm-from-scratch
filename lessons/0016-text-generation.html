从概率分布到文字

训练好的模型输出的是 logits（50257 个分数的向量），不是文字。 你需要一个 generate() 函数，把 logits 转换为 token，再把 token 解码为人类可读的文本。

概念一：generate 函数 —— 逐 token 生成

核心流程——一个 token 一个 token 地循环：

输入 prompt
"人工智能"
→
模型预测
logits → softmax
→
选一个 token
采样策略
→
拼上去
再输入模型
→
重复
直到够长或遇到结束符
@torch.no_grad()
def generate(model, tokenizer, prompt, max_new_tokens=50,
             temperature=0.8, top_k=50, device='cpu'):
    """给定 prompt，生成后续文本"""
    model.eval()

    # 1. 把 prompt 编码成 token IDs
    input_ids = tokenizer.encode(prompt)
    input_ids = torch.tensor([input_ids]).to(device)  # (1, L)

    generated = input_ids.tolist()[0]

    for _ in range(max_new_tokens):
        # 2. 前向传播 → logits
        logits = model(input_ids)              # (1, seq_len, vocab_size)
        logits = logits[:, -1, :]              # 只取最后一个位置的预测
        logits = logits / temperature          # Temperature 缩放

        # 3. Top-K 过滤：只保留分数最高的 K 个 token
        if top_k > 0:
            topk_vals, _ = torch.topk(logits, top_k)
            threshold = topk_vals[:, -1:]      # 第 K 大的值
            logits[logits < threshold] = float('-inf')

        # 4. Softmax → 采样
        probs = torch.softmax(logits, dim=-1)
        next_token = torch.multinomial(probs, num_samples=1)  # 按分布随机选

        # 5. 拼到序列末尾
        input_ids = torch.cat([input_ids, next_token], dim=-1)
        generated.append(next_token.item())

        # 6. 遇到结束符就停
        if next_token.item() == tokenizer.eos_token_id:
            break

    return tokenizer.decode(generated)

# 使用示例
output = generate(model, tokenizer, "人工智能的未来是",
                  max_new_tokens=100, temperature=0.8, top_k=50)
print(output)

概念二：三种采样策略

策略代码效果
Temperaturelogits / TT<1 更确定（保守），T>1 更随机（有创意）
Top-Ktopk(logits, K)只从分数最高的 K 个中选，过滤低分噪声
Top-P (Nucleus)累加到 P 的 token只从累积概率达 P 的最小集合中选，动态适应

概念三：你的第一个生成结果

训练后，试试不同的 prompt 和参数组合：

# 试试这些 prompt:
prompts = [
    "人工智能的未来是",          # 开放式
    "今天天气真好，",            # 日常
    "根据爱因斯坦的相对论，",     # 知识类（模型大概率胡说八道）
]

for prompt in prompts:
    for temp in [0.5, 0.8, 1.2]:
        output = generate(model, tokenizer, prompt,
                         max_new_tokens=80, temperature=temp)
        print(f"[T={temp}] {output}\n")
🎯 预期结果与下一步 你的第一个模型生成的文本可能：
• 语法基本通顺但内容荒谬（正常——小数据+小模型的必然结果）
• 过度重复某个词或短语（叫「重复退化」，增加 temperature 或 top_k 可缓解）
• 中文夹杂乱码 token（GPT-2 tokenizer 对中文不友好，正常现象）

下一步建议：
① 换一个更大的中文语料（如中文维基子集）
② 换成中文专用模型架构（如 Qwen-0.5B 的配置）
③ 或者——直接跳到 Phase 4，理解这些现象背后的原因
