🧱 第5课 · Transformer Block —— 一层完整的处理单元

第 4 课讲了单头 Attention。本课回答：一层完整的 Transformer 处理单元里，除了 Attention 还需要什么？为什么？

⚠️ 本课前提 需要知道单头 Attention 的原理（第 4 课）：Q/K/V、注意力权重、加权求和。本课在此基础上加入三个新组件，拼成完整 Block。

什么是「Block」？—— 深度学习的建筑砖块

在深入具体组件之前，先理解一个基础概念：在深度学习中，「Block」（块）就是一个可复用的处理单元。

输入向量 → [ Block ] → 输出向量

设计好一个 Block 的内部结构，然后把它复制 N 份，首尾相连叠在一起：

输入 → [ Block₁ ] → [ Block₂ ] → [ Block₃ ] → …… → [ Blockₙ ] → 输出

关键点：所有 Block 结构相同（都是「先 Attention 再 FFN」），但参数独立（Block₁ 的权重和 Block₂ 的是不同的数字，各自学习）。GPT-2 小型叠了 12 个，GPT-3 叠了 96 个。

现在的问题是：一个 Block 里面应该包含哪些操作？

类型需求对应组件
功能让 token 之间交换信息（且不止一个视角）Multi-Head Attention（§1）
功能对每个 token 做非线性加工Feed-Forward Network（§2）
工程保证堆了很多层之后梯度不消失残差连接（§3）
工程防止数值范围在多层之后漂移失控Layer Normalization（§4）

§1 Multi-Head Attention —— 不止一个视角

为什么一个头不够？

第 4 课讲的单头 Attention 只用了一组 (WQ, WK, WV)。这意味着每个词只能从一个角度去看其他词。

但语言是立体的。看这句话：

「小明把小红的花瓶打碎了，他很生气。」

要理解「他」指的是谁（小明还是小红？），需要同时追踪：

  • 语义关系：「打碎」的主语是谁？（语义头）
  • 句法结构：「他」在从句中的语法位置？（句法头）
  • 距离关系：「他」和「小明」「小红」各隔了几个词？（位置头）

一个 Attention 头不可能同时从这三个角度去匹配——因为一组 WQ/WK/WV 只能编码一种匹配规则。

做法：多组并行的 Attention

不只有一组 (WQ, WK, WV)，而是有 h 组（GPT-2 用 12 个头）：

🏷️
语义
🔗
语法
📍
位置
😊
情感
⋯
……
头₁：  Attention(QW₁ᴼ, KW₁ᴷ, VW₁ⱽ) → 可能学会了「主语-谓语」关系
头₂：  Attention(QW₂ᴼ, KW₂ᴷ, VW₂ⱽ) → 可能学会了「代词-指代对象」关系
……
头₁₂：每个头产出一个小向量，最后拼接 → 乘 WO 合并回原始维度

每个头的运算和单头完全一样，只是用的 W 矩阵不同——初始值不同，训练后自然分化出不同功能。

🔑 为什么多头比单头好？ 单头 = 一个人用一只眼睛看世界，只能看到一种信息。
多头 = 12 个人从不同角度同时看同一句话，有的看语法，有的看语义，有的看指代——综合起来信息量大得多。

§2 Feed-Forward Network（FFN）—— 逐位置的非线性加工

Attention 有一个本质局限

注意力（Attention）的输出是多个 Value 向量的加权平均。加权平均只能产生输入向量的线性组合。

什么是线性？什么是非线性？

一个变换是线性的，意味着画在坐标系里是一条直线：

线性：  y = 2x + 3    → 画出来是直线
非线性：y = x²         → 画出来是抛物线（弯的）
        y = max(0, x)  → 在 x=0 处有一个折角

这和 Attention 有什么关系？Attention 做加权平均 → 线性操作。它的输出永远被限制在输入向量的线性组合范围内——不能创造出性质完全不同的新表示。

举例：Attention 聚合了「不」和「好看」两个词的向量。加权平均得到的是两者的中间状态——有「不」的属性也有「好看」的属性，但不是「丑」。要得到「不好看 ≈ 丑」，需要一个非线性变换来把「不 + 好看」映射到完全不同的方向。

这就是 FFN 存在的理由：引入非线性。否定、条件、转折、类比——这些语言中的核心逻辑关系，都是非线性的。

具体结构

FFN(x) = GELU( x · W₁ + b₁ ) · W₂ + b₂

三步操作：升维（768→3072）→ 非线性激活（GELU）→ 降维（3072→768）。

为什么「升维再降维」？中间的 3072 维给了模型一个更大的临时计算空间。在低维空间中线性不可分的数据，投影到高维后可能变得可分。这是一种用冗余换表达能力的设计。

关键性质：逐位置独立——FFN 对每个 token 分别计算，不同 token 之间不交换信息。交换信息是 Attention 的工作。FFN 只负责「拿到 Attention 聚合好的上下文之后，怎么加工这个 token 自己的表示」。

🧪 实验 1：线性 vs 非线性 —— 亲眼看到「掰弯」

一组排成网格的点。点「线性变换」：点动了，但网格线依然是直的。点「+ ReLU」：负半轴的点被压扁到 y=0，形状被「掰弯」了。这就是 Attention（线性）和 FFN（非线性）的本质区别。

§3 残差连接（Residual Connection）—— 保证深层网络能训练

问题：层数多了，梯度回不去了

当 Block 堆到 12 层、96 层时，出现一个严重的工程问题：反向传播时，梯度从最后一层传回第一层，每穿过一个操作就衰减一点。到了前面几层，梯度几乎为零——参数无法更新。

这叫梯度消失。数学原因：链式法则让 N 个局部导数连乘，每个局部导数通常 ≤ 1。一大堆 ≤ 1 的数连乘，乘得越多，结果越接近 0。

每个因子平均 0.5，12 层 → 0.5¹² ≈ 0.00024
每个因子平均 0.5，96 层 → 0.5⁹⁶ ≈ 10⁻²⁹（基本是 0）

结果：前面几层的参数收不到有效的调整信号 → 前面的层卡住不动，模型实际上只有最后几层在学。

⚠️ 梯度接近 0 ≠ 训练好了
高中数学里「令导数 = 0，解出极值点」是对的——但那是假设函数是一只光滑的碗。神经网络的损失函数不是碗，而是一片有无数山峰、高原、峡谷的巨大山脉。梯度消失意味着信号在路上衰减完了，不是到了谷底。

判断标准不是梯度大小，而是损失值（Loss）是否还在下降。损失是终极裁判，梯度只是指路的向导。

解决方案：加一条「快捷通道」

output = x + Sublayer(x)

关键在于 + x：原始输入 x 被原封不动地加到子层的输出上。反向传播时，对 x 求导：

∂(output)/∂x = 1 + ∂(Sublayer)/∂x

那个 1 是关键的——即使 Sublayer 内部的梯度衰减到接近零，至少有 1 的梯度能直接传回前面的层。这条不经过任何变换的直通路径，解决了梯度消失问题。

另一个角度：有残差连接时，Sublayer 不需要从零学习完整变换。它只需要学「当前输入离理想输出还差多少」——即残差。如果 Sublayer 什么都没学到（输出全零），output = x + 0 = x，至少不比原来更差。

🧪 实验 2：梯度衰减模拟器 —— 感受链式法则的威力

模拟梯度从最后一层传回第一层。每根柱子代表一层的梯度大小。调整层数和衰减因子，观察前面几层的梯度如何指数级缩水。勾选「残差连接」，看 +1 怎么拯救一切。

第 1 层梯度：0.00024
衰减倍数：~4100×

§4 Layer Normalization（层归一化）—— 防止数值失控

问题：数值漂移

数据每穿过一个子层（Attention 或 FFN），向量里各个维度的数值范围就可能发生漂移。经过多次矩阵乘法和非线性激活后，某些维度的值可能变得极大，另一些变得极小。如果不加控制，几层之后激活函数饱和、梯度消失或爆炸。

LayerNorm 在每次子层计算之前，把输入强行拉回标准范围。

对于一个 token 的 768 维向量，LayerNorm 做三步：

计算均值和方差：求这 768 个数值的平均值 μ 和方差 σ²
标准化：每个值减去 μ，除以 √(σ² + ε)，得到均值为 0、方差为 1 的向量
缩放和平移：乘以 γ（可学习），加上 β（可学习），让模型有能力调整分布
LayerNorm(x) = γ · (x - μ) / √(σ² + ε) + β

为什么 NLP 用 LayerNorm 而不是 BatchNorm？因为文本序列长度不一致，同一个 batch 里不同样本的 padding 长度不同，跨 batch 做统计没有意义。LayerNorm 对每个 token 独立计算自己的均值和方差，不受序列长度和 padding 的影响。

🧪 实验 3：数值漂移模拟器 —— 为什么需要「拉回来」

模拟一个 20 维向量连续穿过多个层的效果。每点一次「穿过一层」，数值就会漂移——有些维度暴涨，有些趋近于零。勾选 LayerNorm 后，每层都会把数值重新拉回均值为 0、方差为 1 的范围。

已过层数：0
均值：0.00
标准差：1.00
最大值：2.31

组装：一个完整的 Transformer Block

现在把四个组件拼在一起。一个 Block 包含两个子层，每个子层配一个残差连接。LayerNorm 放在每个子层前面（现代做法，Pre-LN）：

输入 x
→
LayerNorm
→
Multi-Head
Attention
→
+
←── x 跳过 Attention 直接过来 ──
→
LayerNorm
→
Feed-Forward
Network
→
+
→ 输出
←── x 跳过 FFN 直接过来 ──

写成伪代码：

def transformer_block(x):
    x = x + multi_head_attention(layer_norm(x))   # 子层 1
    x = x + ffn(layer_norm(x))                     # 子层 2
    return x

Pre-LN vs Post-LN

Post-LN（原始论文，2017）

x → Attention → + x → LN → FFN → + x → LN

LN 在子层后面。层数深时训练不稳定。

Pre-LN（现代做法，GPT-2 起）✅

x → LN → Attention → + x → LN → FFN → + x

LN 在子层前面。训练更稳定，现代 LLM 标配。

叠起来：12 个 Block 串联

Embedding + 位置编码
        ↓
[ Block₁ ]  →  浅层：局部词汇关联
[ Block₂ ]  →  浅层：短语级别模式
[ Block₃ ]
   ……
[ Block₁₂ ] →  深层：全局语义、长距离依赖
        ↓
最终输出 → 预测下一个 token

每个 Block 有自己的参数。12 个 Block 互不共享。数据从头流到尾，每经过一层就被精炼一次。

总结：为什么 Attention 做一次不够
单次 Attention 只能让信息在直接相关的 token 之间流动一次。多层堆叠让信息逐层扩散到更远的位置。同时，每层用多个头从不同角度理解，用 FFN 做非线性变换，用残差连接保证梯度能传回去，用 LayerNorm 防止数值失控。

这就是 Transformer Block 存在的根本原因：一轮 Attention 不够，需要反复做，而且每一轮都需要配套的工程保障。

🧪 小测验

问题 1/3 — 多头注意力的意义

12 个注意力头和 1 个注意力头相比，核心优势是什么？

点击看答案
多头可以从不同「视角」同时理解同一句话。有的头学会关注语法关系，有的关注语义相似性，有的关注指代关系——最后综合起来，比单一视角的信息量大得多。而且不同头之间参数独立，各自在训练中自然分化出不同功能。

问题 2/3 — 残差连接

如果把残差连接去掉，堆 96 层 Transformer 会发生什么？为什么？

点击看答案
训练不动——前面几层的梯度接近于零，参数无法更新。
没有残差连接时，梯度必须穿过每一层内部的所有矩阵乘法和激活函数才能到达前面的层。96 层下来，梯度呈指数级衰减。残差连接的「+ x」提供了梯度绕过子层的直通路径——至少保证有 +1 的梯度能传回去。

问题 3/3 — Attention 和 FFN 的分工

既然 Attention 已经聚合了上下文信息，为什么还需要 FFN？

点击看答案
Attention 只能做线性组合（加权平均），FFN 引入了非线性变换。
Attention 负责「谁和谁相关」（信息路由），FFN 负责「这个 token 的表示怎么变换」（信息加工）。语言中的否定、条件、类比等关系是非线性的，线性组合无法表示。分工：Attention = 交流，FFN = 思考。
