🛠️ 第18课 · 训练技巧

同样的模型，有人训得好有人训得差——区别在这

L14 教会了你训练循环的「骨架」。这节课讲「肌肉」——那些让训练更快、更稳、效果更好的技巧。 这些技巧的重要性不亚于模型架构本身。

技巧一：学习率 Warmup + Cosine Decay

训练刚开始时，参数是随机初始化的——梯度方向可能很「野」。 如果一上来就用满学习率，模型容易被带偏，甚至梯度爆炸。

Warmup 就是「前几百步慢慢加速」：

def get_lr_scheduler(optimizer, warmup_steps=100, total_steps=5000):
    def lr_lambda(step):
        if step < warmup_steps:
            # Warmup: 线性增长
            return step / warmup_steps
        else:
            # Cosine Decay: 平滑下降
            progress = (step - warmup_steps) / (total_steps - warmup_steps)
            return 0.5 * (1 + math.cos(math.pi * progress))

    return torch.optim.lr_scheduler.LambdaLR(optimizer, lr_lambda)
💡 为什么 Warmup+Cosine 是标配？ 这是 GPT-3 论文里用的策略，后来成为几乎所有大模型训练的默认配置。
Warmup 步数通常设为总步数的 1%~5%（比如 5000 步中 warmup 100 步）。

技巧二：梯度裁剪（Gradient Clipping）

某个 batch 恰好在「困难」的数据上——比如句尾出现了一个极其罕见的 token—— 梯度可能会异常大。如果不加限制，这一步更新会把参数拽飞，之前几千步的训练成果都可能被毁掉。 

# 在 loss.backward() 和 optimizer.step() 之间
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)

# max_norm=1.0 是经验值——超过 1.0 就截断

梯度裁剪像「安全带」——正常训练时不触发，但遇到异常时防止毁灭性更新。

技巧三：混合精度训练（FP16/BF16）

默认情况下，PyTorch 用 FP32（32 位浮点数）存储参数和计算。 混合精度训练用 FP16 做矩阵乘法，用 FP32 存参数—— 速度翻倍 + 显存减半，精度几乎不降。 

scaler = torch.amp.GradScaler()  # or torch.cuda.amp.GradScaler()

with torch.autocast(device_type='cuda', dtype=torch.float16):
    logits = model(inputs)
    loss = loss_fn(logits, targets)

scaler.scale(loss).backward()      # 放大 loss 防止梯度下溢
scaler.unscale_(optimizer)         # 还原梯度
clip_grad_norm_(model.parameters(), 1.0)
scaler.step(optimizer)
scaler.update()
精度速度显存精度损失
FP32基准基准无
FP16 混合精度~2×~0.5×极小
BF16 混合精度~2×~0.5×几乎无（范围更安全）
⚠️ 注意 FP16 在老 GPU（如 T4、V100）上效果最好；新 GPU（A100、H100）推荐用 BF16。 如果你的训练速度受限于 GPU 算力（不是数据加载），混合精度是最划算的加速手段。

技巧四：梯度累积 —— 「假装」用大 batch

Batch size 越大训练越稳定——但显存放不下怎么办？梯度累积： 连续跑 N 个小 batch，梯度加起来但不更新，攒够了再一次性更新。 效果等价于 batch size × N，显存占用等于一个小 batch。 

accumulation_steps = 4  # 假装 batch_size 翻了 4 倍

for i, (inputs, targets) in enumerate(dataloader):
    logits = model(inputs)
    loss = loss_fn(logits, targets) / accumulation_steps  # 关键：除以累积步数
    loss.backward()  # 梯度累加，不更新

    if (i + 1) % accumulation_steps == 0:
        optimizer.step()
        optimizer.zero_grad()  # 攒够了，更新 + 清空

技巧五：数据去重与质量过滤

Garbage in, garbage out。 训练数据里有大量重复、低质量的文本（HTML 标签、广告、乱码）， 模型会花很多参数去「记忆」这些噪声。

实践证明：用更少但更高质量的数据训练，效果远超用更多低质量数据。 LLaMA 和 Phi 系列模型的核心竞争力之一就是高质量数据筛选流水线。

  • 去重：删除完全重复或高度相似的段落
  • 语言过滤：如果是中文模型，扔掉纯英文文本
  • 质量过滤：扔掉过短、无意义、乱码的文本
  • 去污染：去掉与测试集/benchmark 重合的数据（防止「作弊」）
